# Feature Spec: Внешние ссылки

## Назначение

Внешние ссылки помогают пользователю перейти с локальной страницы Doctor Amal на подтвержденные внешние платформы доктора.

В текущей версии внешние ссылки являются навигационными кнопками. Они не собирают данные пользователя, не выполняют регистрацию, не запускают оплату и не создают внутренний пользовательский сценарий после перехода.

## Где пользователь видит функцию

Внешние кнопки находятся на единственной странице текущей версии рядом с другими основными кнопками проекта.

В текущей версии должны быть видны все внешние кнопки:

- `Telegram`;
- `WhatsApp`;
- `YouTube`;
- `Instagram`.

Кнопка `Купить` и Telegram-ссылка для связи с менеджером по курсу описаны отдельно в `spec/feature-specs/course-purchase-link.md`.

## Что пользователь видит

Пользователь видит список кнопок с понятными названиями внешних платформ:

- `Telegram`;
- `WhatsApp`;
- `YouTube`;
- `Instagram`.

Каждая активная внешняя кнопка должна вести только на подтвержденный URL соответствующей платформы.

Если URL платформы не подтвержден, кнопка все равно должна быть видна в интерфейсе, чтобы сохранять понятный вид текущей версии. Такая кнопка должна быть неактивной, должна показывать текст `Ссылка будет добавлена позже` и не должна выглядеть как рабочая внешняя ссылка.

## Действия пользователя

Пользователь может:

- увидеть внешние кнопки текущей версии;
- нажать активную внешнюю кнопку, если для нее задан подтвержденный URL;
- перейти на соответствующую внешнюю платформу.

Пользователь не может через внешнюю кнопку:

- отправить данные внутрь проекта;
- оформить покупку внутри проекта;
- оплатить курс внутри проекта;
- получить автоматическую медицинскую консультацию;
- получить обещание доступности внешней платформы.

## Основное поведение

Если URL внешней платформы задан и подтвержден владельцем проекта, при нажатии кнопка открывает эту внешнюю платформу в новой вкладке.

Если URL не задан, соответствующая внешняя кнопка остается видимой, но должна быть неактивной с текстом `Ссылка будет добавлена позже`. Она не должна выглядеть как рабочая ссылка и не должна обещать переход на внешнюю платформу.

Проект не должен придумывать URL внешних платформ в specs, коде или UI.

Проект не отвечает за содержание, доступность, правила работы, блокировки, авторизацию или технические ограничения внешних платформ после перехода пользователя по ссылке.

## URL внешних платформ

- URL каждой платформы необязателен;
- активная кнопка обязана иметь подтвержденный URL;
- активная кнопка открывает URL в новой вкладке;
- до подтверждения точного URL внешняя кнопка остается видимой, но неактивной.

## Ограничения текущей версии

В текущей версии внешние ссылки не являются:

- интеграцией с внешними платформами;
- авторизацией через внешнюю платформу;
- формой заявки;
- онлайн-оплатой;
- аналитикой переходов;
- серверной проверкой доступности платформы;
- автоматической подпиской пользователя;
- гарантией доступа к внешнему ресурсу.

Внешние кнопки являются только переходами на подтвержденные URL.

## Исключительные ситуации

Если URL внешней платформы не задан, соответствующая кнопка должна оставаться видимой, но неактивной.

Если внешняя платформа недоступна из-за отсутствия интернета, ограничений устройства, блокировок, авторизации или правил самой платформы, локальная страница не должна показывать это как техническую ошибку проекта.

Если пользователь ожидает, что Doctor Amal сможет управлять содержанием, доступом или ошибками внешней платформы, интерфейс не должен создавать такое ожидание.

Если внешняя платформа изменила URL, правила доступа или формат ссылок, это требует обновления specs перед изменением реализации.

## Обязательные состояния

- все четыре внешние кнопки видимы;
- кнопка активна только при заданных `url` и `is_enabled = true`;
- активная кнопка использует `target="_blank"` и `rel="noreferrer"`;
- кнопка без рабочего URL неактивна;
- текст неактивной внешней кнопки: `Ссылка будет добавлена позже`.

## Что не входит в текущую версию

В текущую версию не входят:

- аналитика кликов по внешним ссылкам;
- отслеживание источников переходов;
- UTM-метки, если они отдельно не подтверждены;
- авторизация через Telegram, WhatsApp, YouTube или Instagram;
- отправка сообщений в WhatsApp или Telegram из проекта;
- автоматическая подписка на канал или аккаунт;
- проверка доступности внешних платформ;
- серверная интеграция с внешними платформами;
- сбор имени, телефона, email, Telegram username или других данных пользователя.

Любая такая возможность является новым требованием и требует отдельного подтверждения владельца проекта, обновления specs и только после этого реализации.

## Будущие функции

Аналитика, UTM-метки, проверка доступности и интеграции с внешними платформами не входят в текущую версию. Будущие расширения ведутся в `spec/technical-specs/future-extension-plan.md`.
