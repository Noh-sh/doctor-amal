# Feature Spec: Админка для редактирования контента

## Назначение

Админка нужна для будущей онлайн-версии Doctor Amal, чтобы доктор мог самостоятельно редактировать подтвержденный контент публичной Taplink-страницы без изменения файлов проекта разработчиком.

Этот feature spec описывает поведение будущей админки. Он не добавляет оплату, заявки, покупателей, медицинские анкеты, аналитику или личный кабинет.

## Где доктор видит функцию

Админка должна быть доступна только авторизованному доктору через отдельный защищенный раздел.

Предварительный маршрут будущей реализации:

```text
/admin
```

Страница `/admin` не является публичным пользовательским маршрутом. Обычный посетитель Taplink-страницы не должен видеть вход в админку как часть публичного сценария.

## Роль

На первом этапе админки контент редактирует только доктор.

Менеджер, помощник, покупатель или обычный пользователь не получают доступ к редактированию контента на первом этапе.

## Вход

Подтвержденный способ входа на первом этапе:

```text
email/password
```

Вход нужен только для доктора. Он не создает личный кабинет для покупателя и не добавляет авторизацию обычных пользователей.

## Что доктор может редактировать

На первом этапе доктор может редактировать только подтвержденный публичный контент страницы:

- краткое представление доктора;
- текстовые блоки профиля доктора;
- внешние ссылки `Telegram`, `WhatsApp`, `YouTube`, `Instagram`;
- активность внешней ссылки;
- курсы;
- название курса;
- описание курса;
- цену курса текстом;
- активность курса;
- общую Telegram-ссылку менеджера для кнопки `Купить`.

Изменения публикуются сразу после сохранения.

## Фото доктора

Фото доктора может редактироваться через админку, когда в этом появится необходимость.

На первом минимальном этапе редактирование фото не является обязательной частью админки, потому что оно требует отдельной технической проверки:

- где хранится файл;
- какие форматы разрешены;
- какой максимальный размер файла;
- как показывается предпросмотр;
- как заменяется старое фото;
- как не допустить загрузку неподходящего изображения.

Если редактирование фото включается в первый этап реализации, это должно быть отдельно описано в technical specs до кода.

## Что доктор не может редактировать на первом этапе

На первом этапе через админку нельзя редактировать:

- медицинское предупреждение `Информация на странице носит ознакомительный характер.`;
- структуру публичной страницы;
- список публичных кнопок сверх подтвержденных;
- route-структуру проекта;
- RLS policies;
- Auth users;
- технические id и ключи таблиц;
- seed или migration-файлы;
- системные env-переменные.

Медицинское предупреждение не редактируется через админку на первом этапе, чтобы не менять медицинский смысл без отдельной проверки.

## Публикация изменений

На первом этапе нет режима черновиков.

После сохранения изменения сразу становятся опубликованными и могут появиться на публичной странице после обновления данных и кеша.

Режим `Черновик -> Предпросмотр -> Опубликовать` может быть добавлен позже отдельным требованием.

## Журнал изменений

На первом этапе журнал изменений не нужен.

Если позже потребуется видеть, кто и когда изменил цену, описание или ссылку, нужно создать отдельный spec для audit log.

## Основное поведение

1. Доктор открывает защищенный раздел админки.
2. Если доктор не вошел, он видит форму входа.
3. Доктор вводит email и пароль.
4. После успешного входа доктор видит разделы редактирования контента.
5. Доктор меняет нужные поля.
6. Доктор нажимает `Сохранить`.
7. Админка сохраняет изменения в Supabase.
8. Публичная страница `/` читает обновленные данные из Supabase.

## Ошибки и ограничения

Если доктор не авторизован, админка не показывает формы редактирования.

Если пользователь не является доктором, он не получает доступ к редактированию.

Если сохранение не удалось, админка должна показать понятную ошибку без технического stack trace.

Если ссылка не задана, внешняя кнопка на публичной странице остается видимой, но неактивной.

Если Telegram-ссылка менеджера не задана, кнопка `Купить` остается неактивной.

## Что не входит

В первый этап админки не входят:

- регистрация покупателей;
- личный кабинет покупателя;
- заявки;
- формы покупки;
- онлайн-оплата;
- заказы;
- платежные статусы;
- автоматическая выдача доступа;
- медицинские анкеты;
- хранение персональных медицинских данных;
- аналитика;
- рассылки;
- роль менеджера;
- журнал изменений;
- черновики и предпросмотр публикации.

## Связь с публичной страницей

Админка не должна менять публичный пользовательский сценарий Taplink-страницы.

Публичная страница остается:

- одним маршрутом `/`;
- с блоками `О докторе` и `Курсы`;
- с внешними кнопками;
- с кнопкой `Купить`, ведущей в Telegram к менеджеру при подтвержденной ссылке;
- без форм, оплаты и сбора пользовательских данных.
