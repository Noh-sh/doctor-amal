# Feature Spec: Админка для редактирования контента

## Назначение

Админка нужна для текущей online-версии Doctor Amal, чтобы доктор мог самостоятельно редактировать подтвержденный контент публичной Taplink-страницы без изменения файлов проекта разработчиком.

Этот feature spec описывает текущее поведение админки. Он не добавляет оплату, заявки, покупателей, медицинские анкеты, аналитику или личный кабинет.

## Где доктор видит функцию

Админка должна быть доступна только авторизованному доктору через отдельный защищенный раздел.

Текущий маршрут:

```text
/admin
```

Страница `/admin` не является публичным пользовательским маршрутом. Обычный посетитель Taplink-страницы не должен видеть вход в админку как часть публичного сценария.

## Роль

На первом этапе админки контент редактирует только доктор.

Менеджер, помощник, покупатель или обычный пользователь не получают доступ к редактированию контента на первом этапе.

## Вход

Подтвержденный способ входа на первом этапе:

```text
email/password
```

Вход нужен только для доктора. Он не создает личный кабинет для покупателя и не добавляет авторизацию обычных пользователей.

Если доктор забыл пароль, он должен иметь возможность начать восстановление прямо со страницы `/admin`.

Восстановление пароля:

- доступно через действие `Забыли пароль?` на форме входа;
- просит ввести email доктора;
- отправляет письмо восстановления через Supabase Auth;
- показывает нейтральное сообщение `Если email зарегистрирован, ссылка для восстановления отправлена.`;
- не сообщает, найден email или нет;
- не создает регистрацию, личный кабинет, покупателей или новые роли;
- не дает доктору доступ к Supabase Dashboard.

После перехода по recovery link доктор задает новый пароль и затем возвращается к входу в `/admin`.

## Что доктор может редактировать

На первом этапе доктор может редактировать только подтвержденный публичный контент страницы:

- краткое представление доктора;
- текстовые блоки профиля доктора;
- внешние ссылки `Telegram`, `WhatsApp`, `YouTube`, `Instagram`;
- активность внешней ссылки;
- курсы;
- добавление нового курса;
- название курса;
- описание курса;
- цену курса текстом;
- активность курса;
- общую Telegram-ссылку менеджера для кнопки `Купить`.

Изменения сохраняются в Supabase сразу после нажатия `Сохранить`. Публичная страница может показать обновленные данные примерно в течение 30 секунд из-за `revalidate: 30`.

Если доктор редактирует медицински значимый текст профиля или курса, перед сохранением он вручную проверяет формулировку по `medical-content-rules.md`.

Текущая админка не выполняет автоматическую медицинскую модерацию и не подтверждает медицинский смысл текста за доктора.

## Добавление нового курса

Доктор может добавить новый курс через админку. `insert`-доступ для таблицы `courses` разрешен только активному `doctor_admin`.

При создании курса доктор заполняет:

- название курса;
- описание курса;
- цену курса текстом;
- подтверждена ли цена;
- показывать ли курс на публичной странице.

Новый курс использует общую Telegram-ссылку менеджера для кнопки `Купить`, как и остальные курсы.

Новый курс не получает:

- отдельную страницу;
- отдельную Telegram-ссылку;
- отдельную форму заявки;
- оплату внутри проекта;
- автоматическую выдачу доступа.

Если новый курс создается активным, он сохраняется в Supabase сразу после нажатия `Сохранить`. Публичная страница может показать его в блоке `Курсы` примерно в течение 30 секунд из-за `revalidate: 30`.

Если новый курс создается неактивным, он остается в админке, но не показывается на публичной странице.

## Фото доктора

Фото доктора не редактируется через текущую админку.

Загрузка или замена фото является будущей функцией и требует отдельного feature spec, technical spec, Storage policies и work plan.

## Что доктор не может редактировать на первом этапе

На первом этапе через админку нельзя редактировать:

- медицинское предупреждение `Информация на странице носит ознакомительный характер.`;
- структуру публичной страницы;
- список публичных кнопок сверх подтвержденных;
- route-структуру проекта;
- RLS policies;
- Auth users;
- технические id и ключи таблиц;
- seed или migration-файлы;
- системные env-переменные.

Медицинское предупреждение не редактируется через админку на первом этапе, чтобы не менять медицинский смысл без отдельной проверки.

## Скрытие курса вместо удаления

На первом этапе админка не удаляет курсы физически из Supabase.

Если доктор хочет убрать курс с публичной страницы, курс скрывается через состояние `is_active = false`.

Скрытый курс:

- не показывается на публичной странице;
- не оставляет пустой блок или пустое место в списке курсов;
- остается доступен в админке для повторного включения или редактирования.

Физическое удаление записи курса из Supabase требует отдельного требования, отдельной `delete` policy и отдельной проверки риска случайной потери данных.

## Публикация изменений

На первом этапе нет режима черновиков.

После успешного сохранения изменения сразу записываются и считаются опубликованными в Supabase. Публичная страница может показать обновленные данные примерно в течение 30 секунд из-за `revalidate: 30`.

Для медицински значимых текстов сохранение означает, что доктор вручную проверил текст по `medical-content-rules.md` и считает его допустимым для публичной Taplink-страницы.

Текущая версия не выполняет мгновенный сброс кеша после сохранения.

Режим `Черновик -> Предпросмотр -> Опубликовать` не входит в текущую версию.

## Журнал изменений

На первом этапе журнал изменений не нужен.

Audit log не входит в текущую версию.

## Основное поведение

1. Доктор открывает защищенный раздел админки.
2. Если доктор не вошел, он видит форму входа.
3. Если пароль забыт, доктор использует `Забыли пароль?` и задает новый пароль через письмо Supabase Auth.
4. Доктор вводит email и пароль.
5. После успешного входа доктор видит разделы редактирования контента.
6. Доктор меняет нужные поля.
7. Если текст медицински значимый, доктор вручную сверяет его с `medical-content-rules.md`.
8. Доктор нажимает `Сохранить`.
9. Админка сохраняет изменения в Supabase.
10. Публичная страница `/` может показать обновленные данные примерно в течение 30 секунд из-за `revalidate: 30`.

Если на шагах входа, проверки роли или сохранения Supabase/Auth недоступны, доктор видит понятный feedback и не получает доступ к редактору до успешной проверки доступа.

## Ошибки и ограничения

Если доктор не авторизован, админка не показывает формы редактирования.

Если пользователь не является доктором, он не получает доступ к редактированию.

Если доктор вводит неверный пароль, админка показывает понятное сообщение об ошибке входа.

Если доктор запрашивает восстановление пароля, интерфейс не раскрывает, зарегистрирован ли введенный email.

Если сохранение не удалось, админка должна показать понятную ошибку без технического stack trace.

Если Supabase или Supabase Auth недоступны, админка должна показать понятное русское сообщение о временной недоступности входа или сохранения. В этом состоянии формы редактирования контента не показываются, пока приложение не сможет подтвердить сессию и роль `doctor_admin`.

Админка не использует локальный fallback как источник данных для сохранения. Локальный fallback нужен только публичной странице `/` и не означает, что доктор может редактировать контент offline.

Если доктор сомневается, проходит ли медицински значимый текст правила `medical-content-rules.md`, текст нельзя публиковать через `/admin` до ручного уточнения формулировки.

Если ссылка не задана, внешняя кнопка на публичной странице остается видимой, но неактивной.

Если Telegram-ссылка менеджера не задана, кнопка `Купить` остается неактивной.

## Что не входит

В первый этап админки не входят:

- регистрация покупателей;
- личный кабинет покупателя;
- регистрация доктора через публичный UI;
- управление Auth users через UI;
- заявки;
- формы покупки;
- онлайн-оплата;
- заказы;
- платежные статусы;
- автоматическая выдача доступа;
- медицинские анкеты;
- автоматическая медицинская модерация текстов;
- внешняя медицинская проверка текстов;
- хранение персональных медицинских данных;
- аналитика;
- рассылки;
- роль менеджера;
- физическое удаление курсов из Supabase;
- журнал изменений;
- черновики и предпросмотр публикации.

## Связь с публичной страницей

Админка не должна менять публичный пользовательский сценарий Taplink-страницы.

Публичная страница остается:

- одним маршрутом `/`;
- с блоками `О докторе` и `Курсы`;
- с внешними кнопками;
- с кнопкой `Купить`, ведущей в Telegram к менеджеру при подтвержденной ссылке;
- без форм, оплаты и сбора пользовательских данных.
