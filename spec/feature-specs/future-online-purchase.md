# Feature Spec: Будущая онлайн-покупка

## Назначение

Этот spec фиксирует границу между первой локальной версией Doctor Amal и возможной будущей онлайн-покупкой курса внутри проекта.

Первая локальная версия не содержит онлайн-покупку. В ней кнопка `Купить` переводит пользователя в Telegram для связи с менеджером.

Будущая онлайн-покупка описана только как возможное направление развития. Она не является текущим требованием и не является обязательной функцией будущей онлайн-версии.

## Где пользователь видит функцию

В первой локальной версии пользователь не видит онлайн-покупку внутри проекта.

Пользователь видит:

- блок `Курсы`;
- раскрытое описание выбранного курса;
- кнопку `Купить`;
- переход в Telegram для связи с менеджером, если Telegram-ссылка подтверждена.

Онлайн-покупка внутри проекта может появиться только в будущей версии после отдельного подтверждения владельца проекта, обновления specs и отдельной реализации.

## Что пользователь видит в первой локальной версии

В первой локальной версии пользователь не должен видеть:

- форму покупки;
- корзину;
- checkout;
- ввод имени, телефона, email, Telegram username или платежных данных;
- выбор способа оплаты;
- оплату картой;
- сообщение об успешной покупке внутри проекта;
- автоматическую выдачу доступа в закрытый Telegram-канал курса.

Если пользователь нажимает `Купить`, текущий подтвержденный сценарий первой версии — переход в Telegram к менеджеру.

## Возможный будущий сценарий

В будущей онлайн-версии может появиться покупка курса внутри проекта, если в этом будет необходимость и ситуация проекта этого потребует.

Возможный будущий сценарий может включать:

- выбор курса;
- ввод данных пользователя;
- оплату курса картой или другим подтвержденным способом;
- подтверждение успешной оплаты;
- переход или доступ в закрытый Telegram-канал курса после успешной оплаты.

Этот список описывает только возможное направление. Он не является разрешением на реализацию в первой локальной версии.

## Что нужно подтвердить перед будущей реализацией

Перед любой реализацией онлайн-покупки нужно отдельно подтвердить и описать в specs:

- нужна ли онлайн-покупка вообще;
- какие курсы можно покупать внутри проекта;
- какие данные пользователя собираются;
- какие поля формы обязательны;
- какой платежный провайдер используется;
- как обрабатывается успешная оплата;
- как обрабатывается ошибка оплаты;
- как пользователь получает доступ к курсу или закрытому Telegram-каналу;
- какие юридические тексты, согласия, условия оплаты и возврата нужны;
- где хранятся данные покупки;
- нужна ли серверная часть;
- нужна ли база данных;
- нужна ли админка или личный кабинет.

Без отдельного подтверждения этих пунктов онлайн-покупку нельзя реализовывать.

## Ограничения первой локальной версии

В первой локальной версии запрещено добавлять:

- онлайн-оплату;
- оплату картой;
- форму покупки;
- форму заявки на покупку;
- корзину;
- checkout;
- сбор персональных данных;
- сбор платежных данных;
- серверную часть для покупок;
- базу данных покупок;
- личный кабинет;
- админку;
- автоматическую выдачу доступа;
- статус `покупка завершена` внутри проекта.

Эти функции являются новыми требованиями и требуют отдельного подтверждения владельца проекта.

## Связь с кнопкой "Купить"

Кнопка `Купить` в первой локальной версии не является платежной кнопкой.

Она не должна:

- запускать оплату;
- открывать форму покупки;
- создавать заказ;
- сохранять данные пользователя;
- подтверждать покупку;
- автоматически выдавать доступ.

В первой локальной версии кнопка `Купить` остается внешним переходом в Telegram для связи с менеджером.

Если позже будет подтверждена онлайн-покупка, поведение кнопки `Купить` нужно будет заново описать в specs перед изменением реализации.

## Исключительные ситуации

Если пользователь ожидает оплату внутри проекта, интерфейс первой локальной версии не должен создавать ожидание, что такая оплата доступна.

Если владелец проекта захочет добавить форму покупки, оплату, сбор данных или автоматическую выдачу доступа, это нужно считать новым требованием.

Если будущая онлайн-покупка начнет противоречить текущему Telegram-сценарию, перед реализацией нужно обновить связанные specs и явно выбрать новый основной сценарий покупки.

## Что не входит в первую локальную версию

В первую локальную версию не входят:

- покупка курса внутри проекта;
- онлайн-оплата;
- платежная интеграция;
- платежные статусы;
- форма покупки;
- сбор имени, телефона, email, Telegram username или платежных данных;
- хранение покупок;
- серверный API покупки;
- автоматический доступ к закрытому Telegram-каналу;
- письма, уведомления или сообщения после оплаты;
- личный кабинет покупателя;
- админка для управления покупками.

## Связь с будущей онлайн-версией

Будущая онлайн-покупка может стать частью будущей онлайн-версии только после отдельного подтверждения владельца проекта.

Даже в будущей онлайн-версии она не считается обязательной по умолчанию. Если в ней не будет необходимости, проект может продолжать использовать переход в Telegram к менеджеру.

Перед переносом этой идеи в реализацию нужно создать или обновить feature specs, user stories и technical specs для данных, маршрутов, платежей, доступа, ошибок, безопасности и проверки.
