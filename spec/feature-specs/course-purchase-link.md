# Feature Spec: Кнопка "Купить"

## Назначение

Кнопка `Купить` помогает пользователю перейти от выбранного курса к связи с менеджером в Telegram.

В первой локальной версии кнопка не оформляет покупку внутри проекта. Она является внешним переходом для дальнейшего общения с менеджером.

## Где пользователь видит кнопку

Кнопка `Купить` находится в блоке `Курсы` на единственной странице первой локальной версии.

Пользователь видит кнопку только после раскрытия конкретного курса. Сначала в свернутом виде курса видны название курса и цена курса.

## Что пользователь видит

После нажатия на название курса пользователь видит:

- описание курса;
- кнопку `Купить`.

Если Telegram-ссылка для связи с менеджером подтверждена, кнопка `Купить` может быть активной внешней ссылкой.

Если Telegram-ссылка не подтверждена, кнопку нельзя показывать как рабочую активную ссылку. Ее можно показывать только как неактивную кнопку с понятным состоянием, например:

- `Ссылка будет добавлена позже`;
- `Покупка временно недоступна`.

Финальный текст неактивного состояния должен быть подтвержден перед реализацией.

## Действия пользователя

Пользователь может:

- открыть блок `Курсы`;
- раскрыть конкретный курс;
- увидеть описание курса и кнопку `Купить`;
- нажать кнопку `Купить`, если задана подтвержденная общая Telegram-ссылка;
- перейти в Telegram для связи с менеджером.

В первой локальной версии пользователь не может:

- оформить покупку внутри проекта;
- заполнить форму покупки внутри проекта;
- оплатить курс внутри проекта;
- передать через проект имя, телефон, email, Telegram username, медицинские данные или платежные данные;
- автоматически получить доступ в закрытый Telegram-канал курса.

## Основное поведение

Если Telegram-ссылка задана и подтверждена владельцем проекта, при нажатии на `Купить` пользователь должен перейти в Telegram для связи с менеджером.

В первой локальной версии используется одна общая Telegram-ссылка на одного менеджера для всех курсов.

При переходе в Telegram должно передаваться название выбранного курса, чтобы менеджеру было понятно, какой курс интересует пользователя.

Кнопка `Купить` не должна:

- создавать заказ внутри проекта;
- показывать форму заявки;
- запускать онлайн-оплату;
- сохранять данные пользователя;
- показывать сообщение, что покупка завершена;
- обещать автоматический доступ в закрытый Telegram-канал.

Покупка, условия оплаты, доступ к курсу и дальнейшая коммуникация происходят за пределами локальной страницы Doctor Amal.

## Telegram-ссылка

Конкретная Telegram-ссылка должна быть предоставлена или отдельно подтверждена владельцем проекта перед реализацией.

Нельзя придумывать Telegram-ссылку в specs, коде или UI.

Перед реализацией нужно отдельно подтвердить:

- точную Telegram-ссылку для связи с менеджером;
- технический способ передачи названия выбранного курса в Telegram;
- должна ли ссылка открываться в текущей вкладке, новой вкладке или по стандартному поведению браузера.

Общая ссылка для всех курсов и передача названия выбранного курса являются подтвержденным поведением первой версии.

Точный технический формат ссылки или сообщения нельзя придумывать без отдельного подтверждения перед реализацией.

## Ограничения первой локальной версии

В первой локальной версии кнопка `Купить` не является:

- платежной кнопкой;
- формой заявки;
- корзиной;
- checkout-сценарием;
- автоматической выдачей доступа;
- внутренним маршрутом покупки;
- подтверждением покупки.

Кнопка `Купить` является только внешним переходом в Telegram для связи с менеджером.

## Исключительные ситуации

Если Telegram-ссылка не задана, кнопка `Купить` должна быть неактивной и не должна выглядеть как рабочий внешний переход.

Если Telegram недоступен из-за отсутствия интернета, ограничений устройства или правил самой платформы, локальная страница не должна показывать это как техническую ошибку проекта.

Если пользователь ожидает оплату внутри проекта, интерфейс не должен создавать ожидание, что оплата доступна на локальной странице.

Если пользователь ожидает автоматический доступ в закрытый Telegram-канал, интерфейс не должен обещать такой доступ в первой локальной версии.

## Что нужно уточнить перед реализацией

Перед реализацией нужно подтвердить:

- точную Telegram-ссылку;
- технический способ передачи названия выбранного курса в Telegram;
- текст неактивного состояния кнопки без ссылки;
- правило открытия Telegram-ссылки;
- нужны ли дополнительные пользовательские подсказки рядом с кнопкой.

Дополнительное сообщение перед переходом в Telegram не входит в первую локальную версию, пока это отдельно не подтверждено в specs.

## Что не входит в первую локальную версию

В первую локальную версию не входят:

- форма покупки;
- форма заявки;
- сбор имени, телефона, email, Telegram username, медицинских данных или платежных данных;
- онлайн-оплата;
- оплата картой;
- корзина;
- личный кабинет;
- автоматическое подтверждение покупки;
- автоматическая выдача доступа в закрытый Telegram-канал курса;
- серверная часть для покупок;
- база данных покупок.

## Связь с будущей онлайн-версией

В будущей онлайн-версии может появиться покупка курса внутри проекта, если в этом будет необходимость и ситуация проекта этого потребует.

Такое расширение может включать выбор курса, ввод данных, оплату картой и переход или доступ в закрытый Telegram-канал курса после успешной оплаты.

Эта возможность не входит в первую локальную версию. Она не считается обязательной функцией будущей онлайн-версии и может быть реализована только после отдельного подтверждения владельца проекта, обновления specs и отдельной реализации.
