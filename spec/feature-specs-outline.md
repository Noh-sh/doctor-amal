# Материал для будущих feature specs проекта Doctor-Amal

Этот файл собирает детали, которым не место в `functional-map.md`.
`functional-map.md` описывает общую карту проекта, а feature specs должны описывать отдельные функции: поведение, данные, действия пользователя и исключительные ситуации.

## Рекомендуемая структура feature specs

```text
spec/
  feature-specs/
    course-catalog.md
    course-detail-page.md
    requests.md
    doctor-profile.md
    articles.md
    local-data-storage.md
    error-handling.md
```

## Что должно быть в каждой feature spec

- Название функции
- Цель функции
- Где пользователь видит функцию
- Какие данные видит пользователь
- Какие действия доступны пользователю
- Основное поведение функции
- Что сохраняется локально
- Сообщения об успехе
- Ошибки и исключительные ситуации
- Что не входит в текущую локальную версию
- Что может быть добавлено в будущей онлайн-версии

## course-catalog.md

Функция: каталог курсов.

Что нужно описать:
- Список доступных курсов по темам
- Краткое описание каждого курса
- Стоимость курса
- Ключевые проблемы, которые решает курс
- Формат курса: доступ в закрытый Telegram-канал после оплаты
- Описание того, что пользователь получает внутри закрытого Telegram-канала
- Правила добавления пользователя в закрытый Telegram-канал после оплаты
- Статус продаж: открыты, закрыты или скоро откроются
- Период открытия продаж, если он известен
- Правило открытия продаж, например несколько раз в год на ограниченный период
- Кнопка "Подробнее" или "Купить"
- Фильтрация по темам и проблемам
- Поиск по названию курса и симптомам

Исключительные ситуации:
- Пользователь не нашел подходящий курс
- Поиск не дал результатов
- Список курсов пустой
- Данные курса заполнены не полностью

## course-detail-page.md

Функция: страница отдельного курса.

Что нужно описать:
- Полное описание курса
- Цели курса и результаты
- Формат доступа через закрытый Telegram-канал
- Что находится внутри закрытого Telegram-канала
- Как пользователь получает доступ после оплаты
- Открыты ли продажи курса сейчас
- Что происходит после подачи заявки на покупку
- Для кого подходит курс
- Преимущества курса
- Условия покупки
- Кнопка "Приобрести курс"
- Блок "Похожие курсы"

Исключительные ситуации:
- Курс не найден
- У курса нет полной информации
- Похожие курсы отсутствуют
- Пользователь хочет консультацию по подбору курса вместо покупки курса

## requests.md

Функция: заявка на покупку курса или консультацию по подбору курса.

Что нужно описать:
- Единая сущность "заявка"
- Типы заявки: покупка выбранного курса и консультация по подбору курса
- Поля формы: имя, контакт для связи, тип заявки
- Дополнительные поля для покупки курса: выбранный курс, комментарий
- Дополнительные поля для консультации по подбору курса: возраст, основные симптомы, вопросы, выбранные темы
- Валидация полей
- Проверка статуса продаж курса
- Информация о том, что оплата не выполняется на сайте
- Информация о том, что оплата обсуждается с доктором или ассистентом после подачи заявки
- Информация о доступе через закрытый Telegram-канал после согласованной оплаты
- Сообщение о дальнейшем контакте после оформления
- Подтверждение сохранения данных
- Минимизация личных данных

Исключительные ситуации:
- Имя не заполнено
- Контакт для связи не заполнен или заполнен неверно
- Курс не выбран
- Продажи курса закрыты
- При закрытых продажах заявки на покупку не принимаются
- Сайт и Telegram-канал работают в ознакомительном режиме до следующего открытия продаж
- Описание проблемы слишком короткое или пустое
- Заявку не удалось сохранить
- Пользователь пытается оплатить онлайн на сайте, хотя проект не предусматривает прямую онлайн-оплату

## doctor-profile.md

Функция: страница "О докторе".

Что нужно описать:
- Информация о семейном докторе с интегративным подходом
- Описание опыта, специализации и ценностей
- Объяснение, какие проблемы решаются курсами
- Почему нет обратной связи внутри курса
- Как выбрать курс
- Зачем нужна консультация по подбору курса

Исключительные ситуации:
- У пользователя недостаточно информации для выбора курса
- Пользователь хочет задать вопрос доктору напрямую
- Пользователь не понимает разницу между курсом и консультацией по подбору курса

## articles.md

Функция: Telegram-канал / статьи.

Что нужно описать:
- Дублирование контента, публикуемого в Telegram
- Список статей по темам: здоровье, кожа, гормоны, ЖКТ, воспаления, эмоции
- Превью статей с заголовками и кратким описанием
- Возможность открыть полную статью
- Раздел "Последние публикации"

Исключительные ситуации:
- Статей пока нет
- Статья не найдена
- У статьи нет полного текста
- Пользователь хочет перейти из статьи к подходящему курсу

## local-data-storage.md

Функция: локальное хранение данных.

Что нужно описать:
- Где хранятся данные курсов
- Где хранятся заявки
- Как различаются типы заявки: покупка курса и консультация по подбору курса
- Какой формат хранения используется: JSON, SQLite или другой вариант
- Какие данные обязательны
- Какие данные можно редактировать позже
- Как структура должна быть подготовлена к будущему переносу в онлайн-версию

Исключительные ситуации:
- Файл данных отсутствует
- Данные повреждены
- Нет прав на запись
- Сохранение не удалось

## error-handling.md

Функция: обработка ошибок и подсказок.

Что нужно описать:
- Обработка ситуаций без интернета
- Предупреждение о том, что текущая версия работает локально
- Валидация форм и пояснения при неправильном заполнении
- Понятные сообщения об ошибках
- Следующий шаг, который система предлагает пользователю

Исключительные ситуации:
- Нет интернета, но приложение продолжает работать локально
- Поля формы заполнены неверно или частично
- Пользователь не находит подходящий курс
- Пользователь хочет получить консультацию по подбору курса вместо покупки курса
- Система не может сохранить данные локально
