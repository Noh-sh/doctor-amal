# User Stories: Персоны

## Цель

Описать основные типы пользователей, для которых проверяются сценарии текущей Taplink-версии Doctor Amal.

Персоны не являются медицинскими профилями и не используются для диагностики. Они нужны только для проверки удобства интерфейса, понятности переходов и сохранения границ ответственности проекта.

## Persona 1: Пользователь впервые открывает страницу

### Контекст

Пользователь впервые открыл Doctor Amal. Он не знает доктора и хочет быстро понять, кто представлен на странице и куда ведут кнопки.

### Цели

- увидеть фото и имя доктора;
- понять, какие направления доступны;
- перейти к блоку `О докторе`;
- понять, что страница не является медицинской консультацией;
- не столкнуться со сложной навигацией.

### Риски

- пользователь может принять страницу за медицинскую консультацию;
- пользователь может ожидать отдельные страницы или форму связи внутри проекта;
- пользователь может не понять, что часть кнопок ведет на внешние платформы.

## Persona 2: Пользователь интересуется курсами

### Контекст

Пользователь хочет посмотреть, какие курсы доступны, сколько они стоят и какой следующий шаг нужен для покупки.

### Цели

- перейти к блоку `Курсы`;
- увидеть список курсов с названиями и ценами;
- раскрыть интересующий курс;
- прочитать краткое описание курса;
- нажать `Купить`, если ссылка на Telegram менеджера подтверждена.

### Риски

- пользователь может ожидать отдельную страницу курса;
- пользователь может ожидать корзину, форму заявки или онлайн-оплату внутри проекта;
- пользователь может ожидать автоматический доступ в закрытый Telegram-канал.

## Persona 3: Пользователь ищет официальный внешний ресурс доктора

### Контекст

Пользователь пришел на страницу, чтобы перейти в Telegram, WhatsApp, YouTube или Instagram доктора.

### Цели

- увидеть внешние кнопки текущей версии;
- понять, какая кнопка ведет на какую платформу;
- перейти по активной подтвержденной ссылке;
- увидеть понятное неактивное состояние, если URL еще не подтвержден.

### Риски

- пользователь может принять неактивную кнопку за сломанную;
- пользователь может ожидать, что публичная страница управляет доступностью внешней платформы;
- пользователь может ожидать отправку сообщения внутрь проекта.

## Persona 4: Пользователь с медицинским ожиданием

### Контекст

Пользователь может искать медицинскую помощь, объяснение симптомов или персональную рекомендацию.

### Цели

- понять, что страница имеет образовательный, ознакомительный и навигационный характер;
- увидеть короткое медицинское ограничение `Информация на странице носит ознакомительный характер.`;
- не получить ложное ожидание диагноза, лечения или экстренной помощи;
- при серьезных, острых или опасных симптомах понять необходимость очной медицинской помощи.

### Риски

- пользователь может ожидать диагностику;
- пользователь может ожидать персональный совет;
- пользователь может принять описание курса или доктора за схему лечения.

## Persona 5: Доктор как редактор контента

### Контекст

В текущей версии доктор имеет защищенную админку `/admin` для редактирования разрешенного контента. Публичные пользователи не имеют личного кабинета, форм и онлайн-оплаты.

### Цели

- сохранить публичную Taplink-страницу простой;
- безопасно редактировать разрешенный контент через `/admin`;
- не смешивать текущую Taplink-страницу с будущей онлайн-покупкой;
- понимать, какие решения нужно подтвердить перед реализацией;
- не добавлять преждевременно оплату, сбор данных, серверную часть или автоматический доступ.

### Риски

- преждевременно добавить функции будущей онлайн-версии;
- начать собирать персональные данные без specs;
- добавить оплату или интеграции без отдельного подтверждения.
