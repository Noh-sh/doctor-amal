# User Stories: Journey Checklist

## Цель

Проверить реальные пути пользователя после реализации первой локальной Taplink-страницы.

Этот checklist помогает найти:

- лишние шаги;
- пропущенные состояния;
- неправильные кнопки;
- несогласованные тексты;
- сценарии, где пользователь заходит в тупик;
- места, где интерфейс обещает больше, чем разрешено specs.

## Проверка первого знакомства

- Пользователь открывает одну страницу Doctor Amal.
- Пользователь видит фото доктора.
- Пользователь видит имя доктора.
- Пользователь видит основные кнопки: `О докторе`, `Курсы`, `Telegram`, `WhatsApp`, `YouTube`, `Instagram`.
- Пользователь понимает, какие кнопки ведут к блокам на этой же странице.
- Пользователь понимает, какие кнопки ведут на внешние платформы.
- Пользователь видит короткое медицинское ограничение: `Информация на странице носит ознакомительный характер.`

## Проверка блока "О докторе"

- Кнопка `О докторе` ведет к блоку на этой же странице.
- Блок `О докторе` не открывает отдельный маршрут.
- В блоке нет формы связи или записи на консультацию.
- В блоке нет сбора имени, телефона, Telegram username, email или медицинских данных.
- Тексты блока не обещают лечение, выздоровление или гарантированный медицинский результат.
- Конкретные данные о докторе не придумываются без подтверждения владельца проекта.

## Проверка блока "Курсы"

- Кнопка `Курсы` ведет к блоку на этой же странице.
- Блок `Курсы` не является отдельным каталогом с маршрутом.
- Сначала видны названия курсов и цены, если они подтверждены.
- Курс можно раскрыть в рамках одной страницы.
- После раскрытия курса пользователь видит описание и кнопку `Купить`.
- Цена не дублируется в раскрытом курсе, если specs не изменены.
- Нет поиска, фильтров, корзины, формы заявки или онлайн-оплаты.

## Проверка кнопки "Купить"

- Кнопка `Купить` видна только после раскрытия конкретного курса.
- При подтвержденной Telegram-ссылке кнопка ведет в Telegram к менеджеру.
- Для всех курсов используется одна общая Telegram-ссылка менеджера.
- Название выбранного курса не передается автоматически в первой версии.
- Пользователь самостоятельно сообщает менеджеру, какой курс его интересует.
- Без подтвержденной Telegram-ссылки кнопка не выглядит как рабочий переход.
- Без подтвержденной Telegram-ссылки кнопка показывает текст `Покупка временно недоступна`.
- Кнопка не создает заказ внутри проекта.
- Кнопка не запускает онлайн-оплату.
- Кнопка не обещает автоматический доступ в закрытый Telegram-канал.

## Проверка внешних кнопок

- Кнопки `Telegram`, `WhatsApp`, `YouTube`, `Instagram` видны в интерфейсе первой версии.
- Активной становится только кнопка с подтвержденным URL.
- Кнопка без URL остается видимой, но неактивной.
- Кнопка без URL показывает текст `Ссылка будет добавлена позже`.
- Внешняя кнопка не собирает данные пользователя.
- Внешняя кнопка не выполняет авторизацию, оплату, подписку или интеграцию.
- Интерфейс не обещает доступность внешней платформы после перехода.

## Проверка медицинских ограничений

- На странице есть короткое медицинское ограничение: `Информация на странице носит ознакомительный характер.`
- Тексты не обещают выздоровление или гарантированный результат.
- Тексты не ставят диагноз.
- Тексты не дают персональные медицинские рекомендации.
- Тексты не раскрывают конкретные схемы лечения как инструкцию для пользователя.
- Тексты не дают инструкции по приему лекарств, добавок или процедур.
- При серьезных, острых или опасных симптомах пользователь направляется к очной медицинской помощи.

## Проверка отсутствия сбора данных

- На странице нет формы заявки.
- На странице нет формы покупки.
- На странице нет поля имени.
- На странице нет поля телефона.
- На странице нет поля Telegram username.
- На странице нет поля email.
- На странице нет медицинской анкеты.
- На странице нет загрузки анализов, документов или файлов.
- Локальная страница не сохраняет обращения пользователя.

## Проверка отсутствия внутренних маршрутов первой версии

- Нет отдельной внутренней страницы `О докторе`.
- Нет отдельного внутреннего каталога курсов.
- Нет отдельных страниц курсов.
- Нет внутренней страницы `Покупка`.
- Нет внутренней страницы `Заявка`.
- Нет внутренней страницы `Консультация`.
- Основные сценарии происходят на одной странице или через внешние платформы.

## Проверка будущей онлайн-покупки

- Онлайн-покупка не показывается в первой локальной версии.
- Нет checkout.
- Нет оплаты картой.
- Нет платежных полей.
- Нет сообщения об успешной покупке внутри проекта.
- Нет автоматической выдачи доступа в закрытый Telegram-канал.
- Будущая онлайн-покупка не реализуется без отдельного подтверждения и обновления specs.

## Проверка отсутствия тупиков

- Пользователь может вернуться от блока `О докторе` к кнопкам или другим блокам.
- Пользователь может вернуться от блока `Курсы` к кнопкам или другим блокам.
- Неактивная внешняя кнопка не выглядит как техническая поломка.
- Неактивная кнопка `Купить` не выглядит как завершенная покупка.
- Если курсы, цены, тексты или URL еще не подтверждены, интерфейс не придумывает данные и не показывает технические детали.
