# User Stories: Journey Checklist

## Цель
Проверить реальные пути пользователя после реализации интерфейса.

Этот checklist помогает найти:
- лишние шаги;
- пропущенные состояния;
- неправильные кнопки;
- несогласованные тексты;
- сценарии, где пользователь заходит в тупик.

## Проверка первого знакомства
- Пользователь с главной страницы может перейти к "О докторе".
- Пользователь с главной страницы может перейти к каталогу.
- Пользователь с главной страницы может перейти к статьям.
- Пользователь понимает, что курсы образовательные.
- Пользователь понимает, что сайт не ставит диагноз.
- Пользователь не видит заявку при закрытых продажах.

## Проверка каталога
- Каталог показывает опубликованные курсы.
- Каталог не показывает черновики.
- Поиск работает.
- Фильтры работают.
- Статус продаж виден на карточке.
- Поиск без результатов не оставляет пользователя без следующего шага.
- При закрытых продажах каталог ведет к статьям, а не к заявке.

## Проверка страницы курса
- Валидный курс открывается.
- Невалидный `courseId` показывает понятную ошибку.
- Черновик не раскрывается пользователю.
- Условия покупки понятны.
- Telegram-доступ объяснен.
- Онлайн-оплата не предлагается.
- При открытых продажах есть заявка на покупку.
- При закрытых продажах заявок нет.
- При `coming_soon` заявок нет.

## Проверка заявки на покупку
- Заявка открывается с выбранным курсом.
- Нельзя отправить без имени.
- Нельзя отправить без контакта.
- Нельзя отправить без согласий.
- Нельзя отправить при закрытых продажах.
- Повторная заявка обрабатывается.
- Ошибка сохранения не очищает форму.
- Успех показывает корректный текст.

## Проверка заявки на консультацию по подбору курса
- Форма доступна только при открытых продажах.
- Возраст обязателен.
- Описание проблемы обязательно.
- Слишком короткое описание не принимается.
- Контакт валидируется.
- Успех не обещает медицинскую консультацию.
- При закрытых продажах прямая ссылка на форму не позволяет отправить заявку.

## Проверка статей
- Список статей открывается.
- Публикуются только опубликованные статьи.
- Статья открывается по валидному `articleId`.
- Невалидный `articleId` показывает понятную ошибку.
- Статья не ведет напрямую к покупке.
- При закрытых продажах статьи остаются доступными.

## Проверка закрытых продаж
- Главная не предлагает заявку.
- Каталог не предлагает заявку.
- Страница курса не предлагает заявку.
- Страница "О докторе" не предлагает заявку.
- Прямая ссылка на форму заявки не позволяет отправить данные.
- Пользователь видит путь к статьям.

## Проверка медицинских ограничений
- Нет автоматической диагностики.
- Нет назначения лечения.
- Нет обещаний результата.
- Нет просьбы загрузить анализы.
- Нет поля для медицинских документов.
- Есть текст о границах образовательного проекта.

## Проверка оплаты и Telegram
- Нет оплаты картой.
- Нет полей банковской карты.
- Нет платежных данных.
- Нет обещания автоматического доступа в Telegram.
- Есть объяснение, что доступ предоставляется вручную после согласованной оплаты.

## Проверка отсутствия тупиков
- На странице ошибки есть путь назад.
- В пустом каталоге есть понятное сообщение.
- В пустом списке статей есть понятное сообщение.
- При ошибке сохранения пользователь понимает следующий шаг.
- После успешной заявки пользователь может вернуться к каталогу или статьям.
