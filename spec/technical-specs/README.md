# Technical Specs проекта Doctor Amal

## Назначение

Эта папка описывает техническую часть текущей online Taplink-версии Doctor Amal с Supabase и защищенной админкой.

Цель technical specs — дать разработчику или AI достаточно технического контекста, чтобы поддерживать публичную Next.js-страницу, Supabase и защищенную админку без добавления функций, которых нет в `global-spec.md`, `functional-map.md`, feature specs и user stories.

## Общая техническая модель текущей версии

- Текущая версия реализуется как online Next.js-приложение и локально запускается для разработки.
- Используется TypeScript и Next.js App Router.
- Публичная часть состоит из одной страницы `/`.
- Защищенная админка доступна по маршруту `/admin`.
- Кнопки `О докторе` и `Курсы` раскрывают accordion-блоки на этой же странице.
- Кнопки `Telegram`, `WhatsApp`, `YouTube`, `Instagram` открывают подтвержденные URL в новой вкладке.
- Кнопка `Купить` открывает Telegram-ссылку менеджера в новой вкладке, если ссылка подтверждена.
- Основной источник контента — Supabase.
- При отсутствии или недоступности Supabase публичная страница использует подтвержденный локальный fallback.
- Контент через админку редактирует только активный `doctor_admin`.
- Пользовательские данные не собираются и не сохраняются.

## Обязательный стек

- Framework: Next.js.
- Language: TypeScript.
- Routing: Next.js App Router.
- UI: React components внутри Next.js.
- Styling: CSS Modules, глобальный CSS или другой легкий подход, совместимый с Next.js.
- Data source текущей реализации: Supabase для публичного контента страницы.
- Fallback data source: подтвержденные локальные данные проекта.

Next.js является основной платформой публичной страницы и админки.

## Что не входит в текущий technical scope

В текущей версии не реализуются:

- отдельные внутренние маршруты `О докторе`, `Курсы`, `Покупка`, `Заявка`, `Консультация`;
- каталог курсов с отдельными страницами;
- статьи;
- поиск и фильтры;
- формы заявок;
- сохранение обращений в `localStorage`;
- серверный API;
- база данных для пользовательских данных, заявок, заказов, оплаты или медицинских данных;
- auth покупателей;
- управление Auth users через админку;
- новые роли кроме `doctor_admin`;
- загрузка фото через админку;
- физическое удаление курсов;
- аналитика;
- онлайн-оплата;
- автоматический доступ в закрытый Telegram-канал;
- сбор имени, телефона, Telegram username, email, медицинских или платежных данных.

Эти возможности могут появиться только в будущей версии после отдельного подтверждения владельца проекта и обновления specs.

## Термины

- **Taplink-страница** — одна публичная страница с кнопками, внутренними блоками и внешними переходами.
- **Блок `О докторе`** — секция на той же странице, а не отдельный маршрут.
- **Блок `Курсы`** — секция на той же странице со списком курсов, ценой в свернутом виде и раскрытием описания.
- **Кнопка `Купить`** — внешняя ссылка в Telegram к менеджеру, а не платежная кнопка.
- **Внешняя кнопка** — кнопка перехода на подтвержденную внешнюю платформу.
- **Неактивная кнопка** — видимая кнопка без рабочего перехода, если точная ссылка еще не подтверждена.
