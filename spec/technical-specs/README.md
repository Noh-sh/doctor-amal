# Technical Specs проекта Doctor Amal

## Назначение

Эта папка описывает техническую часть Taplink-версии Doctor Amal и следующий подтвержденный технический этап: перевод источника контента на Supabase.

Цель technical specs — дать разработчику или AI достаточно технического контекста, чтобы реализовать одну Next.js-страницу и подключать следующий источник контента без добавления функций, которых нет в `global-spec.md`, `functional-map.md`, feature specs и user stories.

Технические спецификации основаны на:

## Состав technical specs

- `architecture.md` — Next.js-архитектура, слои приложения и ограничения первой версии.
- `data-model.md` — локальные модели данных для профиля, курсов, внешних ссылок и настроек страницы.
- `routing-and-ui.md` — единственный маршрут, структура страницы, компоненты и UI-состояния.
- `local-storage.md` — правило отсутствия пользовательского localStorage и допустимые локальные данные проекта.
- `requests-and-validation.md` — отсутствие заявок в первой версии и техническая валидация локального контента.
- `supabase-content-source.md` — первый Supabase-этап: online-источник контента страницы без заявок, оплаты, auth покупателей, заказов и медицинских данных.
- `supabase-dashboard-setup.md` — инструкция и SQL для создания таблиц, RLS policies и seed-контента в online Supabase Dashboard.
- `supabase-mcp-access.md` — правила будущего MCP-доступа к Supabase: inspection-first, без обхода specs и CLI migrations.
- `admin-auth-and-access.md` — требования будущей админки: Supabase Auth, доступ только для доктора, RLS для записи контента и границы без покупателей, оплаты, заявок и медицинских данных.
- `supabase/migrations/` — воспроизводимые Supabase CLI migrations для структуры базы и seed-контента.
- `future-extension-plan.md` — путь к будущей онлайн-версии с CMS/админкой и возможной онлайн-покупкой.
- `change-management.md` — безопасный порядок изменений после завершения этапов.
- `implementation-checklist.md` — критерии готовности и приемочные сценарии.

## Общая техническая модель первой версии

- Первая версия реализуется как локально запускаемое Next.js-приложение.
- Используется TypeScript и Next.js App Router.
- Публичная часть состоит из одной страницы `/`.
- Кнопки `О докторе` и `Курсы` ведут к блокам на этой же странице.
- Кнопки `Telegram`, `WhatsApp`, `YouTube`, `Instagram` ведут на подтвержденные внешние URL.
- Кнопка `Купить` ведет в Telegram к менеджеру, если ссылка подтверждена.
- Данные страницы хранятся в локальных файлах проекта или константах до Supabase-этапа.
- Следующий подтвержденный технический этап может заменить локальный источник контента на Supabase по `supabase-content-source.md`.
- Пользовательские данные не собираются и не сохраняются.

## Обязательный стек

- Framework: Next.js.
- Language: TypeScript.
- Routing: Next.js App Router.
- UI: React components внутри Next.js.
- Styling: CSS Modules, глобальный CSS или другой легкий подход, совместимый с Next.js.
- Data source текущей реализации: локальные данные проекта для профиля, курсов, ссылок и текстов.
- Data source следующего подтвержденного этапа: Supabase только для публичного контента страницы.

Next.js выбран как основной стек, потому что проект может позже перейти к онлайн-версии с CMS, админкой, серверной частью или API без смены общей платформы.

## Что не входит в первую technical scope

В первой локальной версии не реализуются:

- отдельные внутренние маршруты `О докторе`, `Курсы`, `Покупка`, `Заявка`, `Консультация`;
- каталог курсов с отдельными страницами;
- статьи;
- поиск и фильтры;
- формы заявок;
- сохранение обращений в `localStorage`;
- серверный API;
- база данных для пользовательских данных, заявок, заказов, оплаты или медицинских данных;
- авторизация;
- админка;
- аналитика;
- онлайн-оплата;
- автоматический доступ в закрытый Telegram-канал;
- сбор имени, телефона, Telegram username, email, медицинских или платежных данных.

Эти возможности могут появиться только в будущей версии после отдельного подтверждения владельца проекта и обновления specs.

## Термины

- **Taplink-страница** — одна публичная страница с кнопками, внутренними блоками и внешними переходами.
- **Блок `О докторе`** — секция на той же странице, а не отдельный маршрут.
- **Блок `Курсы`** — секция на той же странице со списком курсов, ценой в свернутом виде и раскрытием описания.
- **Кнопка `Купить`** — внешняя ссылка в Telegram к менеджеру, а не платежная кнопка.
- **Внешняя кнопка** — кнопка перехода на подтвержденную внешнюю платформу.
- **Неактивная кнопка** — видимая кнопка без рабочего перехода, если точная ссылка еще не подтверждена.
