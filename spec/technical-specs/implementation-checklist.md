# Technical Spec: Implementation Checklist

## Цель

Дать проверочный список для реализации первой локальной Taplink-версии Doctor Amal.

## Архитектура

- Приложение реализовано на Next.js + TypeScript.
- Используется Next.js App Router.
- Публичный пользовательский маршрут первой версии — `/`.
- Нет маршрутов `/doctor`, `/courses`, `/courses/:courseId`, `/request`, `/articles`, `/articles/:articleId`, `/admin` как пользовательских сценариев первой версии.
- Приложение запускается через стандартный Next.js dev server.
- Production build собирается стандартной командой Next.js.
- Пользовательские тексты на русском языке.
- Технические ошибки не показываются пользователю как stack trace.

## Данные

- Есть модель `TaplinkPageData`.
- Есть модель `DoctorProfile`.
- Есть модель `Course`.
- Есть модель `ExternalLink`.
- Есть модель `PurchaseSettings`.
- Нет моделей `Article`, `UserRequest`, `Order`, `Payment`, `Admin` для первой версии.
- Локальные данные не содержат пользовательских персональных данных.
- Неподтвержденные курсы, цены, URL и биографические факты не придумываются.

## Первый блок

- Пользователь видит фото доктора, если оно подтверждено, или согласованный временный placeholder.
- Пользователь видит имя доктора.
- Пользователь видит краткое представление, если оно подтверждено.
- Пользователь видит медицинское ограничение `Информация на странице носит ознакомительный характер.`
- Пользователь видит кнопки `О докторе`, `Курсы`, `Telegram`, `WhatsApp`, `YouTube`, `Instagram`.

## Блок "О докторе"

- Кнопка `О докторе` ведет к блоку на этой же странице.
- Блок не открывает отдельный маршрут.
- Блок показывает только подтвержденные данные.
- В блоке нет формы связи.
- В блоке нет записи на консультацию.
- В блоке нет сбора персональных или медицинских данных.
- Тексты не обещают лечение, выздоровление или гарантированный результат.

## Блок "Курсы"

- Кнопка `Курсы` ведет к блоку на этой же странице.
- Блок не является отдельным каталогом.
- В свернутом курсе видны название и цена.
- После раскрытия курса видны описание и кнопка `Купить`.
- Цена не дублируется в раскрытом курсе, если specs не изменены.
- Нет поиска и фильтров.
- Нет корзины.
- Нет формы заявки.
- Нет онлайн-оплаты.

## Кнопка "Купить"

- Кнопка `Купить` видна только после раскрытия курса.
- При подтвержденной Telegram-ссылке кнопка ведет в Telegram к менеджеру.
- Используется одна общая Telegram-ссылка менеджера для всех курсов.
- Название выбранного курса не передается автоматически в первой версии.
- Пользователь самостоятельно сообщает менеджеру, какой курс его интересует.
- Без подтвержденной Telegram-ссылки кнопка не выглядит как рабочий переход.
- Без подтвержденной Telegram-ссылки кнопка показывает `Покупка временно недоступна`.
- Кнопка не создает заказ.
- Кнопка не запускает оплату.
- Кнопка не сохраняет данные пользователя.
- Кнопка не обещает автоматический доступ в закрытый Telegram-канал.

## Внешние кнопки

- Кнопки `Telegram`, `WhatsApp`, `YouTube`, `Instagram` видны.
- Активной становится только кнопка с подтвержденным URL.
- Кнопка без URL остается видимой, но неактивной.
- Кнопка без URL показывает `Ссылка будет добавлена позже`.
- Внешняя кнопка не собирает данные пользователя.
- Внешняя кнопка не выполняет авторизацию, подписку, аналитику или интеграцию.
- Недоступность внешней платформы не показывается как ошибка локальной страницы.

## Медицинские ограничения

- На странице есть текст `Информация на странице носит ознакомительный характер.`
- Тексты не обещают выздоровление или гарантированный результат.
- Тексты не ставят диагноз.
- Тексты не дают персональные медицинские рекомендации.
- Тексты не раскрывают схемы лечения как инструкцию.
- Тексты не дают инструкции по приему лекарств, добавок или процедур.

## Отсутствие сбора данных

- Нет формы заявки.
- Нет формы покупки.
- Нет поля имени.
- Нет поля телефона.
- Нет поля email.
- Нет поля Telegram username.
- Нет медицинской анкеты.
- Нет загрузки файлов.
- Нет записи в `localStorage` обращений пользователя.

## Адаптивность и доступность

- Страница работает на ширине от 360px.
- Кнопки удобны для нажатия на мобильном.
- Текст не переполняет контейнеры.
- Интерактивные элементы доступны с клавиатуры.
- Неактивные кнопки различимы не только цветом.
- Изображение доктора имеет корректный `alt`, если используется.

## Тестовые сценарии приемки

### Первый экран

1. Открыть `/`.
2. Проверить фото, имя, кнопки и медицинское ограничение.
3. Проверить, что нет формы и внутренних маршрутов в навигации.

### Блок "О докторе"

1. Нажать `О докторе`.
2. Проверить переход к блоку на той же странице.
3. Проверить отсутствие формы, записи и медицинских обещаний.

### Блок "Курсы"

1. Нажать `Курсы`.
2. Проверить переход к блоку на той же странице.
3. Проверить свернутый курс: название и цена.
4. Раскрыть курс.
5. Проверить раскрытый курс: описание и `Купить`, без дублирования цены.

### Покупка через Telegram

1. Раскрыть курс.
2. Если Telegram-ссылка менеджера подтверждена, нажать `Купить`.
3. Проверить, что переход идет во внешний Telegram.
4. Проверить, что сайт не показывает форму, оплату или успешную покупку.

### Неактивные внешние ссылки

1. Открыть страницу с неподтвержденным URL.
2. Проверить, что кнопка видима, но не активна.
3. Проверить, что кнопка не ведет на придуманный URL.

## Definition of Done

Функция считается готовой, если:

- реализована одна Taplink-страница;
- все основные сценарии из feature specs и user stories работают;
- ограничения первой версии технически соблюдены;
- пустые и неактивные состояния обработаны;
- пользователь не вводится в заблуждение относительно оплаты, медицины, Telegram-доступа и внешних платформ;
- реализация не мешает будущей замене локальных данных на CMS/API.
