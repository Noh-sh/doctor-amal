# Functional Map проекта Doctor-Amal

## Обзор
Этот файл предназначен для описания общей функциональной карты проекта Doctor-Amal. Здесь будут систематизированы все ключевые функции, страницы, сценарии и требования, которые будут добавляться глобально по мере развития проекта.

## Цель
Создать локальный проект, который предоставляет возможность:
- узнать о докторе и его подходе
- ознакомиться с курсами по поддержке здоровья и изменению образа жизни
- приобрести курсы с полными материалами без обратной связи
- подать заявку на консультацию, чтобы эксперт мог рекомендовать подходящий курс
- просмотреть материалы, дублирующие контент Telegram-канала
- корректно обрабатывать ошибки, отключение интернета и неправильное заполнение полей

## Роли пользователей
- Пользователь: просматривает информацию о докторе, курсы, статьи и оставляет локальные заявки
- Владелец проекта / доктор: в будущем управляет курсами, заявками и материалами

## Что получает пользователь
- Полную информацию о докторе, его подходе и методах
- Каталог курсов по проблемам здоровья
- Детальное описание курса и возможности его покупки
- Локальную форму для оформления заказа
- Возможность оставить заявку на консультацию
- Доступ к материалам, которые дублируют Telegram-канал
- Надёжные подсказки и сообщения в случае ошибок

## Как пользователь решает свою проблему
- Пользователь знакомится с доктором и методикой
- Выбирает направление по своей проблеме: кожа, ЖКТ, гормоны, боли, воспаления, дефициты
- Изучает описание курса, его цель, содержание и ожидаемый результат
- Понимает, подходит ли ему курс или нужна предварительная консультация
- При необходимости оставляет локальную заявку, которая сохраняется в проекте

## Основные пользовательские пути
### Сценарий 1: Поиск и покупка курса
1. Открыть главную страницу
2. Перейти в каталог курсов
3. Найти курс по теме или симптомам
4. Открыть страницу курса
5. Оформить заявку на покупку курса
6. Получить подтверждение сохранения данных

### Сценарий 2: Предварительная консультация для подбора курса
1. Открыть страницу "О докторе" или главную страницу
2. Перейти на страницу заявки на консультацию
3. Заполнить форму с симптомами и вопросами
4. Отправить заявку
5. Увидеть подтверждение, что заявка сохранена для дальнейшего подбора курса

### Сценарий 3: Ознакомление с материалами Telegram
1. Перейти на страницу Telegram-канала / статей
2. Выбрать публикацию по теме
3. Открыть статью и прочитать содержание
4. Вернуться к каталогу курсов или заявке на консультацию

## Точки взаимодействия пользователя
### 1. Главная страница
- Общее приветствие и краткое описание Doctor-Amal
- Польза курсов для здоровья женщин
- Блок с основными темами: кожа, ЖКТ, гормональный баланс, головные боли, хроническое воспаление, дефициты, психологическое состояние
- Ссылка на страницу "О докторе"
- Ссылка на каталог курсов
- Ссылка на страницу Telegram-канала / блога
- Кнопка "Оставить заявку на консультацию"

### 2. Страница "О докторе"
- Информация о семейном докторе с интегративным подходом
- Описание опыта, специализации и ценностей
- Объяснение, какие проблемы решаются курсами
- Почему нет обратной связи внутри курса
- Как выбрать курс и зачем нужна предварительная консультация

### 3. Каталог курсов
- Просмотр списка доступных курсов
- Поиск и фильтрация курсов по темам, проблемам и симптомам
- Переход на страницу выбранного курса
- Переход к оформлению заявки на покупку

### 4. Страница курса
- Просмотр подробной информации о выбранном курсе
- Понимание цели курса, ожидаемого результата и условий покупки
- Переход к заявке на покупку курса
- Просмотр связанных или похожих курсов

### 5. Страница покупки / заказа курса
- Форма для оформления покупки курса локально
- Сохранение заявки на покупку в локальной версии проекта
- Подтверждение результата пользователю

### 6. Страница заявки на консультацию
- Форма для локальной заявки на предварительную консультацию
- Сбор общей информации о проблеме пользователя
- Помощь в подборе подходящего курса перед покупкой
- Сохранение заявки в локальной версии проекта
- Подтверждение результата пользователю

### 7. Страница Telegram-канала / статьи
- Раздел с бесплатными полезными материалами от доктора
- Дублирование статей и публикаций из Telegram-канала
- Помощь пользователю в знакомстве с подходом доктора до покупки курса
- Возможность перейти к Telegram-каналу, каталогу курсов или заявке на консультацию

## Основные функции
### A. Управление курсами
- Хранение списка курсов локально
- Категоризация по проблемам здоровья
- Поиск и фильтрация курса

### B. Оформление покупки
- Локальная форма заявки на приобретение
- Сохранение истории заказов локально

### C. Заявка на консультацию
- Форма для сбора информации о пациенте перед подбором курса
- Локальное хранение заявок

### D. Информационные страницы
- Блок "О докторе"
- Блок "Telegram / статьи"
- Информация о подходе доктора, профилактике и поддержке здоровья

### E. Работа при ошибках
- Обработка ситуаций без интернета
- Предупреждение о том, что в данный момент работа возможна только локально
- Отображение понятных сообщений об ошибках

## Типичные ситуации и исключительные случаи
- Нет интернета, но приложение продолжает работать локально
- Поля формы заполнены неверно или частично
- Пользователь не находит подходящий курс по первичному поиску
- Пользователь сомневается, какой курс ему подходит перед покупкой
- Система должна показать понятное сообщение об ошибке и предложить следующий шаг

## Границы первой локальной версии
- Приложение работает полностью локально на компьютере
- Данные курсов и заявки сохраняются локально
- Нет онлайн-оплаты
- Нет личного кабинета
- Нет автоматической отправки заявок доктору
- Нет медицинской диагностики внутри приложения
- Нет реальной интеграции с сервером

## Граница ответственности
- Приложение не ставит диагноз автоматически
- Материалы курсов носят образовательный и информационный характер
- Рекомендации по здоровью должны даваться доктором с учётом состояния пользователя
- При серьёзных, острых или опасных симптомах пользователь должен обратиться за очной медицинской помощью
- Заявка на консультацию помогает передать информацию доктору, но не заменяет очный медицинский осмотр при необходимости

## Будущее развитие
- После завершения локальной версии проект может быть расширен до онлайн-платформы
- В онлайн-версии возможны база данных, личный кабинет, онлайн-оплата и отправка заявок доктору
- Текущая локальная структура должна проектироваться так, чтобы позже её можно было перенести в онлайн-версию
