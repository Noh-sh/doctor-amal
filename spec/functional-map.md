# Functional Map проекта Doctor Amal

## Роль документа

Этот документ описывает функциональную карту текущей online-версии проекта Doctor Amal.

`global-spec.md` фиксирует общий смысл проекта. `functional-map.md` уточняет структуру страницы, список кнопок, поведение пользователя, границы текущей версии и направление будущего развития.

## Общая модель текущей версии

Текущая версия проекта — это online web-страница в формате Taplink с Supabase как источником контента и защищенной админкой для доктора.

Проект состоит из одной страницы. Пользователь открывает страницу, видит фото и имя доктора, выбирает нужную кнопку и переходит к соответствующему направлению на этой же странице или на внешней платформе.

Публичная часть работает как навигационная страница с внутренними accordion-блоками. В проекте есть только один дополнительный текущий маршрут `/admin`, который не входит в публичный пользовательский сценарий.

## Пользователь

Основной пользователь — русскоязычный пользователь женского пола любого возраста, который умеет читать и пользоваться интернетом.

Пользователь может находиться в любой стране мира. Проект должен быть понятен без привязки к конкретному региону, если внешние платформы доступны пользователю.

## Главная цель пользователя

Пользователь хочет быстро найти официальный ресурс доктора и перейти туда, где размещена нужная информация, канал, материал, предложение или способ связи.

Страница должна помогать пользователю без лишних шагов понять:

- кто представлен на странице;
- какие направления доступны;
- куда ведет каждая кнопка;
- что проект не является медицинской консультацией и не обещает выздоровление.

## Главная бизнес-цель

Главная бизнес-цель текущей версии — собрать основные внешние ресурсы доктора в одном месте и направлять пользователя на нужную платформу.

Визуальный порядок кнопок должен поддерживать эту цель. Если владелец проекта позже задаст конкретный бизнес-приоритет, порядок кнопок нужно обновить в specs перед реализацией.

## Структура публичной страницы

Текущая версия состоит из одной страницы без дополнительных внутренних маршрутов.

В начале страницы должны быть:

- фото доктора;
- имя доктора;
- краткое представление, если значение задано в контенте.

Если фото доктора еще не предоставлено, вместо фото можно показывать согласованный временный placeholder. Placeholder не должен использовать реальную фотографию другого человека и не должен создавать впечатление, что это фото доктора.

После первого блока должны идти кнопки текущей версии. Кнопки `О докторе` и `Курсы` раскрывают accordion-блоки на этой же странице. Внешние кнопки открывают подтвержденные URL в новой вкладке.

На странице также должно быть короткое медицинское ограничение: `Информация на странице носит ознакомительный характер.`

## Кнопки текущей версии

Кнопка — основной функциональный элемент проекта.

В текущей версии подтверждены следующие кнопки:

1. `О докторе`
2. `Курсы`
3. `Telegram`
4. `WhatsApp`
5. `YouTube`
6. `Instagram`

Каждая кнопка должна иметь:

- понятное название;
- целевое действие;
- назначение, понятное пользователю до нажатия.

Конкретные URL для внешних платформ должны быть добавлены в specs перед реализацией. Без подтвержденной ссылки внешняя кнопка остается видимой, но неактивной, с текстом `Ссылка будет добавлена позже`.

## Поведение кнопок

### О докторе

Кнопка `О докторе` не ведет на отдельную внутреннюю страницу в текущей версии.

Так как текущая версия состоит из одной страницы, кнопка `О докторе` должна вести к блоку о докторе на этой же странице.

Блок `О докторе` содержит заданную в контенте информацию:

- где доктор учился;
- какие знания, опыт и профессиональные направления есть у доктора;
- с какими темами здоровья и недомоганиями работает доктор;
- какие подходы и форматы помощи доктор может предложить.

Блок `О докторе` не должен обещать лечение, выздоровление или гарантированный медицинский результат. Он не должен превращаться в медицинскую статью, консультацию, диагностику или раздел с персональными медицинскими рекомендациями.

### Курсы

Кнопка `Курсы` ведет к блоку курсов на этой же странице.

Блок `Курсы` показывает для каждого активного курса:

- названия курсов;
- краткое описание каждого курса;
- цену курса;
- кнопку `Купить`.

В свернутом виде курса пользователь видит название курса и цену.

Курс раскрывается через accordion на этой же странице. Повторное нажатие закрывает курс.

При нажатии на конкретный курс пользователь видит описание курса и кнопку `Купить` в рамках этой же страницы. Цена не дублируется в раскрытом курсе, если specs не изменены.

В текущей версии кнопка `Купить` не открывает форму внутри проекта и не запускает онлайн-оплату. Она переводит пользователя в Telegram для связи с менеджером.

В текущей версии кнопка `Купить` ведет на общую Telegram-ссылку менеджера без автоматической передачи названия курса. Пользователь самостоятельно сообщает менеджеру, какой курс его интересует.

Если Telegram-ссылка менеджера не подтверждена, кнопка `Купить` остается неактивной с текстом `Покупка временно недоступна`.

Список курсов, цены и Telegram-ссылка для связи с менеджером должны быть уточнены в feature specs перед реализацией.

### Telegram

Кнопка `Telegram` ведет на Telegram-ресурс доктора.

Точный URL должен быть добавлен перед реализацией.

### WhatsApp

Кнопка `WhatsApp` ведет на WhatsApp-ресурс или чат, подтвержденный владельцем проекта.

Точный URL должен быть добавлен перед реализацией.

### YouTube

Кнопка `YouTube` ведет на YouTube-ресурс доктора.

Точный URL должен быть добавлен перед реализацией.

### Instagram

Кнопка `Instagram` ведет на Instagram-ресурс доктора.

Точный URL должен быть добавлен перед реализацией.

## Внешние платформы

Внешняя платформа — это ресурс за пределами публичной страницы Doctor Amal.

К внешним платформам текущей версии относятся:

- Telegram;
- WhatsApp;
- YouTube;
- Instagram;
- Telegram для связи с менеджером по покупке курса.

Проект не отвечает за содержание, доступность и правила работы внешних платформ после перехода пользователя по ссылке.

Если внешняя платформа недоступна из-за отсутствия интернета, ограничений устройства или правил самой платформы, публичная страница не должна показывать это как техническую ошибку проекта.

## Маршруты

Текущие маршруты:

- `/` — публичная Taplink-страница;
- `/admin` — защищенная админка для активного `doctor_admin`.

Неизвестный внутренний маршрут показывает 404 с возвратом на `/`.

В текущей версии не должно быть отдельных публичных страниц:

- `О докторе`;
- `Курсы`;
- `Статьи`;
- `Заявка`;
- `Покупка`;
- `Консультация`.

Другие маршруты являются новым требованием. Перед реализацией нужно обновить specs.

## Тексты о докторе, курсах и медицинские ограничения

Внутри текущей версии проекта не должно быть медицинских статей, схем лечения, диагностики и персональных медицинских рекомендаций.

Проект может содержать только:

- имя доктора;
- краткое представление доктора или проекта;
- информацию о том, где доктор учился;
- информацию о знаниях, опыте и профессиональных направлениях доктора;
- описание направлений здоровья, с которыми работает доктор, без обещания результата;
- названия курсов;
- краткие описания курсов;
- цены курсов;
- названия кнопок;
- короткое медицинское ограничение;
- технические или навигационные подписи, если они нужны для понимания кнопок.

В текущей версии не должно быть:

- медицинских статей;
- схем лечения;
- советов по симптомам;
- обещаний результата;
- автоматического подбора материалов;
- персональных медицинских рекомендаций;
- текстов, которые могут восприниматься как консультация врача.

## Сбор данных

Текущая версия не собирает и не хранит данные пользователя.

В текущей версии нет:

- формы заявки;
- поля имени;
- поля телефона;
- поля Telegram username;
- поля email;
- медицинской анкеты;
- локального сохранения обращений пользователя;
- отправки данных доктору или ассистенту.

Любой сбор данных является новым требованием и требует отдельного подтверждения владельца проекта, обновления specs и только после этого реализации.

## Основной пользовательский путь

1. Пользователь открывает публичную страницу Doctor Amal.
2. Пользователь видит фото и имя доктора.
3. Пользователь видит список кнопок.
4. Пользователь выбирает нужное направление.
5. Пользователь нажимает кнопку.
6. Пользователь переходит на внешнюю платформу или к подтвержденному блоку на этой же странице.
7. Если пользователь выбирает курс и нажимает `Купить`, проект переводит его в Telegram для связи с менеджером.

## Исключительные ситуации

Если ссылка для внешней кнопки не задана, такая кнопка остается видимой, но неактивной, с текстом `Ссылка будет добавлена позже`. Она не должна выглядеть как рабочая внешняя ссылка и не должна обещать переход на внешнюю платформу.

Если внешняя платформа недоступна, проект не должен обещать, что сможет решить эту проблему внутри публичной страницы.

Если пользователь ожидает медицинскую помощь, проект должен сохранять границу ответственности: страница не заменяет очную консультацию и не оказывает экстренную помощь.

Если требуется собрать данные пользователя, это не должно добавляться в текущую версию без отдельного обновления specs.

## Границы текущей версии

В текущую версию входит:

- публичная online-страница `/`;
- защищенная админка `/admin`;
- Supabase как основной источник контента;
- локальный fallback публичного контента;
- фото доктора;
- имя доктора;
- кнопки `О докторе`, `Курсы`, `Telegram`, `WhatsApp`, `YouTube`, `Instagram`;
- переходы на подтвержденные внешние платформы;
- блок `О докторе` на этой же странице;
- блок `Курсы` на этой же странице;
- названия, описания и цены курсов;
- кнопка `Купить` с переходом в Telegram для связи с менеджером;
- русскоязычный интерфейс;
- короткое медицинское ограничение.

В текущую версию не входит:

- отдельный внутренний каталог курсов;
- внутренние страницы курсов с отдельными маршрутами;
- оформление и оплата покупки курсов внутри проекта;
- формы заявок;
- медицинские статьи, схемы лечения и персональные медицинские рекомендации;
- онлайн-консультация;
- онлайн-оплата;
- auth покупателей;
- личный кабинет;
- управление Auth users через админку;
- загрузка фото через админку;
- физическое удаление курсов;
- автоматическая диагностика;
- персональные медицинские рекомендации внутри проекта;
- сбор и хранение данных пользователя.

## Актуальные детальные specs

Текущее поведение раскрыто в feature specs:

- блок `О докторе`: состав информации, допустимые формулировки, медицинские ограничения;
- блок `Курсы`: структура курса, описание, цена и способ раскрытия информации о курсе;
- кнопку `Купить`: поведение и переход в Telegram к менеджеру;
- внешние ссылки: URL, состояние без ссылки, открытие в новой вкладке;
- медицинские ограничения пользовательских текстов;
- будущую онлайн-покупку как отдельное направление будущего развития.

## Будущее развитие

Будущие функции вынесены в `spec/technical-specs/future-extension-plan.md` и `spec/feature-specs/future-online-purchase.md`.

Они не являются разрешением на реализацию и не меняют текущую online-версию без отдельного подтверждения и обновления актуальных specs.
