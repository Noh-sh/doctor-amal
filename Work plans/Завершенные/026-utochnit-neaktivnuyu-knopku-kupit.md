# План 026: Уточнить неактивную кнопку "Купить"

## Статус

Завершен.

## Цель

Уточнить поведение кнопки `Купить`, если Telegram-ссылка для покупки еще не задана.

## Подтверждение specs

Основание:

- Подтверждение владельца проекта в текущей задаче.
- `spec/feature-specs/courses-block.md`

Подтверждено:

- Формулировка `будущий раздел` не подходит для кнопки `Купить`.
- Если Telegram-ссылка не задана, кнопку `Купить` можно показывать только как неактивную кнопку с понятным состоянием.

Вне specs:

- Конкретный текст состояния не выбран окончательно.
- Ссылка для покупки не добавляется.

## Что делаем

1. Заменить формулировку про `будущий раздел`.
2. Уточнить допустимые состояния неактивной кнопки `Купить`.

## Что не делаем в этом плане

- Не добавляем Telegram-ссылку.
- Не меняем код приложения.
- Не выполняем git commit и push.

## Проверка

Команды:

```bash
rg -n "будущий раздел|неактивн|Ссылка будет добавлена позже|Покупка временно недоступна|Telegram-ссылка" spec/feature-specs/courses-block.md
git diff --check
```

Ручная проверка:

- Убедиться, что для кнопки `Купить` больше нет формулировки `будущий раздел`.
- Убедиться, что неактивное состояние понятно пользователю.

## Критерии готовности

- Правило для кнопки `Купить` без Telegram-ссылки уточнено.
- Кнопка без ссылки не выглядит как рабочий переход.
- Результат проверки записан в план.

## Результат проверки

Выполнены проверки:

```bash
rg -n "будущий раздел|неактивн|Ссылка будет добавлена позже|Покупка временно недоступна|Telegram-ссылка" spec/feature-specs/courses-block.md
git diff --check
```

Результат:

- Для кнопки `Купить` без Telegram-ссылки убрана формулировка `будущий раздел`.
- Уточнено, что кнопку можно показывать только как неактивную кнопку с понятным состоянием.
- Примеры состояний: `Ссылка будет добавлена позже` или `Покупка временно недоступна`.
- `git diff --check` выполнен без ошибок.
