# План 003: Каталог курсов

## Статус

Завершен.

## Цель

Подключить страницу `/courses` к локальным данным курсов и показать пользователю базовый каталог опубликованных курсов.

## Подтверждение specs

Основание:

- `spec/feature-specs/course-catalog.md`
- `spec/technical-specs/routing-and-ui.md`
- `spec/technical-specs/data-model.md`
- `spec/technical-specs/implementation-checklist.md`

Подтверждено:

- каталог должен загружать список опубликованных курсов;
- каталог должен скрывать неопубликованные курсы;
- карточка курса должна показывать название, краткое описание, категорию, темы или симптомы, стоимость, статус продаж, период продаж, если он известен, и кнопку “Подробнее”;
- каталог не должен показывать пользователю технические идентификаторы курса, внутреннее название Telegram-канала или служебные поля;
- каталог не должен предлагать покупку при закрытых продажах;
- при пустом списке нужно показать понятное пустое состояние;
- UI должен использовать repository/service layer, а не читать JSON напрямую.

Вне specs:

- нет.

## Что делаем

1. Подключить `/courses` к `getPublishedCourses()`.
2. Добавить компонент карточки курса.
3. Показать опубликованные курсы из `data/courses.json`.
4. Показать один общий блок состояния продаж над каталогом.
5. Показать статус продаж понятным русским текстом.
6. Показать стоимость или текст “Стоимость уточняется”.
7. Показать период продаж, если он указан.
8. Показать объяснение Telegram-доступа.
9. Добавить пустое состояние для отсутствия опубликованных курсов.

## Что не делаем в этом плане

- Не реализуем поиск.
- Не реализуем фильтры.
- Не реализуем страницу курса с реальными данными.
- Не реализуем кнопку “Купить” в каталоге.
- Не реализуем форму заявки.
- Не реализуем сохранение заявок.
- Не добавляем серверный API.
- Не меняем `spec/`.

## Проверка

Команды:

```bash
npm run build
```

Ручная проверка:

- открыть `/courses`;
- проверить, что видны только опубликованные курсы;
- проверить, что черновой курс не виден;
- проверить, что сверху есть один общий блок состояния продаж;
- проверить, что у каждого курса есть статус продаж;
- проверить, что при закрытых продажах нет кнопки покупки;
- проверить, что нет внутреннего названия Telegram-канала;
- проверить, что кнопка “Подробнее” ведет на `/courses/:courseId`.

## Критерии готовности

- `/courses` показывает опубликованные курсы из repository.
- Черновики и архивные записи не видны.
- Карточки курсов отличаются от временной заглушки и содержат реальные данные.
- Есть общий блок состояния продаж.
- Статусы продаж показаны текстом.
- Закрытые продажи не предлагают покупку.
- Build проходит.
