# 040. Исправить раскрытие блоков Taplink-страницы

## Статус

Завершено.

## Цель

Сделать раскрытие блоков `О докторе` и `Курсы` устойчивым после обновления ветки и перезапуска dev-сервера.

## Основание в specs

- `spec/functional-map.md`: кнопки `О докторе` и `Курсы` ведут к блокам на этой же странице.
- `spec/feature-specs/doctor-block.md`: блок `О докторе` может раскрываться на одной странице без отдельного маршрута.
- `spec/feature-specs/courses-block.md`: блок `Курсы` раскрывает список курсов через accordion на этой же странице.
- `spec/technical-specs/routing-and-ui.md`: допустимо раскрытие секции на одной странице без дополнительных маршрутов.

## Что сделано

- Раскрытие верхних блоков `О докторе` и `Курсы` переведено на native `details` / `summary`.
- Убран клиентский `useState` из `TaplinkPage`, чтобы раскрытие не зависело от React state и stale client bundle.
- Стили открытого состояния обновлены под `[open]`.
- Случайное изменение `next-env.d.ts` от dev/build-режима не включается в изменение задачи.

## Проверка

- `npm run build` выполнен успешно после изменения раскрытия.
- `git diff --check` выполнен успешно.
- Локально проверено, что главная страница содержит native `details` / `summary` для блоков `О докторе` и `Курсы`.

## Результат

Блоки `О докторе` и `Курсы` раскрываются прямо под соответствующими кнопками на одной Taplink-странице и остаются в рамках specs первой версии.
