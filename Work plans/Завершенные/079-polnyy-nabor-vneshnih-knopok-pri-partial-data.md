# План 079: Полный набор внешних кнопок при partial-data

## Статус

Завершен.

## Цель

Исправить partial-data сценарий публичной страницы: если Supabase возвращает не все строки `external_links`, пользователь все равно должен видеть все внешние кнопки текущей версии в безопасном неактивном состоянии.

## Подтверждение specs

Основание:

- `Work plans/Активные/068-master-plan-do-10-iz-10.md`;
- `spec/technical-specs/supabase-content-source.md`;
- `spec/technical-specs/implementation-checklist.md`;
- `spec/technical-specs/change-management.md`.

Подтверждено:

- пользователь видит кнопки `Telegram`, `WhatsApp`, `YouTube`, `Instagram`;
- внешняя кнопка без URL остается видимой, но неактивной;
- активной становится только кнопка с подтвержденным URL;
- URL нельзя придумывать или автоматически исправлять;
- ошибки данных не должны приводить к белому экрану или stack trace.

Вне specs:

- новые внешние платформы;
- добавление URL, UTM или аналитики;
- изменение Supabase schema, RLS или seed-контента;
- изменение пользовательского сценария.

## Что делаем

1. При маппинге Supabase `external_links` начинать с локального подтвержденного набора четырех внешних кнопок.
2. Поверх него применять только валидные строки Supabase по известным платформам.
3. Если строка платформы отсутствует или URL невалиден, кнопка остается видимой и неактивной.
4. Не менять визуальный текст и публичный сценарий.

## Проверка

Команды:

```bash
npm run quality
git diff --check
```

Ручная логика:

- при полном наборе Supabase rows отображаются все 4 кнопки;
- при неполном наборе Supabase rows недостающие платформы остаются видимыми и неактивными;
- невалидный URL не активирует кнопку.

## Критерии готовности

- Публичная страница всегда получает 4 внешние кнопки текущей версии.
- Недостающие или невалидные Supabase rows не создают активные ссылки.
- `npm run quality` проходит.
- `git diff --check` проходит.

## Результат проверки

- В `data/taplink-page-source.ts` добавлен mapper `mapExternalLinks`.
- Публичная страница теперь получает полный локально подтвержденный набор внешних кнопок: `Telegram`, `WhatsApp`, `YouTube`, `Instagram`.
- Supabase rows переопределяют данные только для известных платформ.
- Если строка платформы отсутствует, кнопка остается видимой и неактивной из локального безопасного fallback.
- Если URL невалиден, кнопка остается видимой и неактивной.
- Новые платформы, URL, UTM, migrations, Supabase данные и пользовательский сценарий не менялись.
- `npm run quality` выполнен успешно.
- `git diff --check` выполнен без ошибок.
