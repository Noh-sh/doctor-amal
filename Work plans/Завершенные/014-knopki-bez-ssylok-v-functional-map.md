# План 014: Кнопки без ссылок в functional map

## Статус

Завершен.

## Цель

Уточнить правило для внешних кнопок без подтвержденной ссылки: такую кнопку можно показывать, но нельзя показывать как активную рабочую ссылку.

## Подтверждение specs

Основание:

- Подтверждение владельца проекта в текущей задаче.
- `spec/functional-map.md`

Подтверждено:

- Кнопку без ссылки можно показывать в интерфейсе, если она нужна как будущий раздел или неактивный элемент.
- Такая кнопка не должна выглядеть как рабочая внешняя ссылка и не должна обещать переход.

Вне specs:

- Новые кнопки и ссылки не добавляются.

## Что делаем

1. Обновить формулировку про внешние кнопки без ссылок в `spec/functional-map.md`.
2. Проверить, что документ не запрещает показывать кнопку без ссылки вообще.

## Что не делаем в этом плане

- Не меняем код приложения.
- Не добавляем реальные ссылки.
- Не выполняем git commit и push.

## Проверка

Команды:

```bash
rg -n "неактивн|будущий раздел|рабочая внешняя ссылка|обещать переход|Без подтвержденной ссылки" spec/functional-map.md
git diff --check
```

Ручная проверка:

- Убедиться, что кнопку без ссылки можно показывать как неактивную или будущий раздел.
- Убедиться, что кнопку без ссылки нельзя показывать как рабочий переход.

## Критерии готовности

- Правило не запрещает сам показ кнопки без ссылки.
- Правило запрещает вводить пользователя в заблуждение рабочим видом ссылки.
- Результат проверки записан в план.

## Результат проверки

Выполнены проверки:

```bash
rg -n "неактивн|будущий раздел|рабочая внешняя ссылка|обещать переход|Без подтвержденной ссылки|нельзя показывать" spec/functional-map.md
git diff --check
```

Результат:

- В `functional-map.md` больше нет полного запрета на показ кнопки без ссылки.
- Кнопку без ссылки можно показывать как неактивную кнопку или будущий раздел.
- Кнопка без ссылки не должна выглядеть как рабочая внешняя ссылка и не должна обещать переход.
- `git diff --check` выполнен без ошибок.
