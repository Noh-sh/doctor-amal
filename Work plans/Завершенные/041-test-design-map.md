# План 041: Тестовая design map

## Статус

Завершено.

## Цель

Создать тестовую design map, чтобы сначала проверять визуальные решения Taplink-страницы в отдельном рабочем документе и только после подтверждения переносить их в настоящий дизайн.

## Подтверждение specs

Основание:

- `spec/functional-map.md`
- `spec/technical-specs/routing-and-ui.md`
- `spec/technical-specs/change-management.md`

Подтверждено:

- первая версия является одной Taplink-страницей;
- финальное расположение блоков может уточняться при UI-реализации, если не меняются сценарии и не добавляется новое поведение;
- интерфейс должен работать на мобильной ширине от 360px, планшете и десктопе;
- элементы должны быть доступными с клавиатуры, без переполнения текста и горизонтального скролла;
- внешние кнопки и кнопка `Купить` должны сохранять подтвержденные specs состояния.

Вне specs:

- новый продуктовый сценарий не добавляется;
- тестовая design map не является утвержденным финальным дизайном.

## Что делаем

1. Создать отдельный документ тестовой design map.
2. Зафиксировать в нем границы тестирования дизайна.
3. Добавить checklist для мобильной, планшетной и десктопной проверки.
4. Зафиксировать правило: перенос в настоящий дизайн только после ручного подтверждения.

## Что не делаем в этом плане

- Не меняем UI-код.
- Не добавляем маршруты, формы, оплату, заявки или сбор данных.
- Не утверждаем реальные фото, курсы, цены или URL.
- Не переносим тестовые визуальные решения в основной дизайн без отдельного подтверждения.

## Проверка

Команды:

```bash
git diff --check
```

Ручная проверка:

- убедиться, что документ не добавляет новое поведение сверх specs;
- убедиться, что документ описывает именно тестирование дизайна до внедрения.

## Критерии готовности

- Создана тестовая design map.
- В документе есть границы, проверочные состояния и критерии переноса в настоящий дизайн.
- План завершен, проверен и перенесен в `Work plans/Завершенные/`.
- Обновлены `Roadmap/chronology.md` и `Roadmap/project-roadmap.md`.

## Результат проверки

- `git diff --check` выполнен без ошибок.
- Ручная проверка документа выполнена: test design map не добавляет новые маршруты, формы, оплату, сбор данных, медицинские сценарии или неподтвержденные факты.
- Документ фиксирует правило переноса в настоящий дизайн только после ручного подтверждения владельца проекта.
