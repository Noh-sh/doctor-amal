# План 044: План действий перед Supabase

## Статус

Завершен.

## Цель

Подготовить пошаговый план перехода к Supabase как источнику контента страницы до начала разработки и подключения базы данных.

## Подтверждение specs

Основание:

- `spec/technical-specs/architecture.md`
- `spec/technical-specs/data-model.md`
- `spec/technical-specs/future-extension-plan.md`
- `spec/technical-specs/change-management.md`

Подтверждено:

- первая локальная версия была без базы данных, server API, админки, авторизации, форм и онлайн-оплаты;
- будущая онлайн-версия может заменить локальные данные на CMS/API без изменения публичного пользовательского сценария;
- перед расширением нужно сначала подтвердить требование, обновить specs и только потом менять код;
- Supabase для текущего следующего этапа должен рассматриваться только как источник контента страницы, а не как заявки, оплата, пользователи или медицинские данные.

Вне specs:

- код Supabase в этом плане не реализуется;
- база данных не подключается;
- таблицы не создаются;
- пользовательский сценарий не меняется.

## Что делаем

1. Создать документ плана действий до разработки Supabase.
2. Зафиксировать порядок: specs → online Supabase Dashboard → env-переменные → подключение данных → проверки → deploy.
3. Разделить, что входит в первый Supabase-этап, а что остается за пределами.
4. Подготовить основу для следующего плана `045-supabase-content-specs`.

## Что не делаем в этом плане

- Не устанавливаем Supabase CLI.
- Не устанавливаем `@supabase/supabase-js`.
- Не создаем `supabase/`.
- Не меняем код приложения.
- Не создаем таблицы.
- Не добавляем админку, авторизацию, заявки, оплату или сбор данных пользователя.

## Проверка

Команды:

```bash
git diff --check
```

Ручная проверка:

- план не добавляет реализацию;
- план не противоречит первой версии и future extension plan;
- следующий шаг ясно отделяет specs от разработки;
- план фиксирует работу сразу через online Supabase Dashboard, без локального Supabase CLI/Docker этапа.

## Критерии готовности

- Создан документ с пошаговым планом до разработки Supabase.
- В плане ясно указано, какие specs нужно создать или обновить.
- В плане указано, что сначала Supabase хранит только контент страницы.
- Проверка выполнена и записана в план.

## Результат проверки

- `git diff --check` выполнен без ошибок.
- Ручная проверка выполнена: план не добавляет реализацию, не подключает базу данных и не меняет пользовательский сценарий.
- Следующий шаг отделен от разработки: сначала `045-supabase-content-specs`, потом online Supabase Dashboard, env-переменные и код.
- Уточнено, что локальный Supabase через Docker/CLI не используется на первом этапе.
- Добавлен порядок работы с ключами: Project URL, publishable key или legacy anon key, `.env.local`, hosting env, запрет secret/service role key во frontend.
- План закрыт как подготовительный: последующие Supabase-этапы `045`-`051` уже выполнены и проверены отдельными планами.
- `044-supabase-action-plan.md` перенесен в завершенные как справочный документ к этому подготовительному этапу.
