# План 097: Тестовый дизайн по референсу

## Статус

Завершен.

## Цель

Сделать тестовый визуальный вариант публичной Taplink-страницы Doctor Amal по предоставленному референсу: с приглушенным зеленым фоном, светлой центральной карточкой и более крупным фото из текущей аватарки.

## Подтверждение specs

Основание:

- `spec/global-spec.md`;
- `spec/functional-map.md`;
- `spec/feature-specs/doctor-block.md`;
- `spec/feature-specs/courses-block.md`;
- `spec/technical-specs/routing-and-ui.md`;
- `spec/technical-specs/change-management.md`;
- `Design map/test-design-map.md`;
- `Design map/design-rules.md`.

Подтверждено:

- можно тестировать композицию, размер фото, отступы, цвета, карточки и адаптивность;
- дизайн должен сохранять green nutrition направление, одну страницу `/`, существующие кнопки и accordion-блоки;
- фото доктора может быть крупным и мягко скругленным;
- тестовый дизайн не требует изменения specs, если не меняет продуктовый сценарий.

Вне specs:

- новых функций, текстов, данных, маршрутов и пользовательских сценариев нет.

## Что делаем

1. Перестроить визуальную композицию публичной страницы по референсу.
2. Увеличить текущую аватарку почти до полной ширины карточки.
3. Уточнить фон, поверхности, кнопки и раскрытые блоки.
4. Проверить адаптивность и качество сборки.

## Что не делаем в этом плане

- Не меняем данные и пользовательские тексты.
- Не меняем поведение accordion и внешних ссылок.
- Не меняем админку, Supabase и specs.
- Не выполняем commit или push.

## Проверка

Команды:

```bash
npm run quality
git diff --check
```

Ручная проверка:

- фото заметно крупнее прежнего;
- страница сохраняет мобильную Taplink-композицию;
- на ширине 360px нет горизонтального скролла;
- раскрытые блоки и disabled-состояния остаются читаемыми.

## Критерии готовности

- Тестовый дизайн соответствует референсному направлению и текущим specs.
- Проверки проходят.
- Результат проверки записан в план.
- План перенесен в завершенные, Roadmap обновлен.

## Результат проверки

- `npm run quality` выполнен успешно: прошли `lint`, `typecheck` и production-сборка Next.js.
- `git diff --check` выполнен без ошибок.
- Локальный Next.js-сервер запущен, публичная страница `/` вернула HTTP `200`.
- Для ширины 360 px проверен расчет layout: центральная карточка имеет доступную ширину 340 px, внутренний контент и фото остаются в ее границах без горизонтального переполнения.
- Автоматический визуальный скриншот не выполнен: в рабочем окружении нет headless-браузера.
