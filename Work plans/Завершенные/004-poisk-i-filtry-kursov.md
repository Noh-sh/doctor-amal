# План 004: Поиск и фильтры курсов

## Статус

Завершен.

## Цель

Добавить на страницу `/courses` интерактивный поиск и фильтры по опубликованным курсам без изменения бизнес-логики покупки.

## Подтверждение specs

Основание:

- `spec/feature-specs/course-catalog.md`
- `spec/technical-specs/routing-and-ui.md`
- `spec/technical-specs/data-model.md`
- `spec/technical-specs/implementation-checklist.md`

Подтверждено:

- каталог должен поддерживать поиск по названию курса, категории, проблеме, симптому и теме здоровья;
- технически поиск должен использовать `title`, `shortDescription`, `category`, `topics`, `symptoms`;
- каталог должен поддерживать фильтр по теме или категории;
- каталог должен поддерживать фильтр по статусу продаж;
- при поиске без результатов нужно предложить изменить запрос;
- поиск не должен интерпретировать симптом как диагноз;
- каталог не должен предлагать покупку при закрытых продажах.

Вне specs:

- нет.

## Что делаем

1. Добавить client component для интерактивного каталога.
2. Добавить поисковую строку.
3. Добавить фильтр по категории.
4. Добавить фильтр по статусу продаж.
5. Добавить состояние “ничего не найдено”.
6. Сохранить карточки курсов и общий блок состояния продаж.

## Что не делаем в этом плане

- Не реализуем страницу курса с реальными данными.
- Не реализуем кнопку покупки.
- Не реализуем форму заявки.
- Не реализуем сохранение заявок.
- Не добавляем медицинскую диагностику.
- Не добавляем серверный API.
- Не меняем `spec/`.

## Проверка

Команды:

```bash
npm run build
```

Ручная проверка:

- открыть `/courses`;
- ввести запрос, который находит курс по названию;
- ввести запрос, который находит курс по теме или симптому;
- выбрать категорию;
- выбрать статус продаж;
- проверить состояние без результатов;
- убедиться, что черновики не появляются;
- убедиться, что поиск не формулирует диагноз.

## Критерии готовности

- Поиск работает по названию, описанию, категории, темам и симптомам.
- Фильтр категории работает.
- Фильтр статуса продаж работает.
- При пустом результате есть понятное сообщение.
- Черновые курсы не появляются.
- Build проходит.
