# План 043: UI design rules

## Статус

Завершено.

## Цель

Зафиксировать UI-правила проекта Doctor Amal, чтобы дальнейшие визуальные правки делались последовательно и опирались на утвержденный дизайн.

## Подтверждение specs

Основание:

- `spec/functional-map.md`
- `spec/technical-specs/routing-and-ui.md`
- `Design map/test-design-map.md`

Подтверждено:

- первая версия остается одной Taplink-страницей;
- визуальные детали можно уточнять, если не меняются сценарии, маршруты, данные, медицинский смысл и бизнес-правила;
- test design map уже фиксирует правило: перенос тестовых решений в настоящий дизайн только после ручного подтверждения владельца проекта;
- владелец проекта подтвердил перенос текущего визуального варианта в реальный дизайн.

Вне specs:

- UI design rules не добавляют новые функции;
- документ фиксирует подтвержденные UI-правила первой Taplink-версии.

## Что делаем

1. Создать `Design map/design-rules.md`.
2. Зафиксировать палитру, шрифт, фон, кнопки, блоки, иконки, glass-эффекты и mobile-first правила.
3. Отдельно описать тестовый и утвержденный дизайн.
4. Зафиксировать, что UI-правки должны читать этот документ перед изменениями.
5. Перевести статус правил в утвержденные после подтверждения владельца проекта.

## Что не делаем в этом плане

- Не меняем UI-код.
- Не добавляем новые маршруты, формы, оплату, заявки или сбор данных.
- Не утверждаем реальные фото, курсы, цены или URL.

## Проверка

Команды:

```bash
git diff --check
```

Ручная проверка:

- документ не добавляет новое поведение сверх specs;
- документ описывает только визуальные правила и границы их применения.

## Критерии готовности

- `Design map/design-rules.md` создан.
- В документе есть основные UI-правила проекта.
- Проверка выполнена и записана в план.

## Результат проверки

- `git diff --check` выполнен без ошибок.
- Ручная проверка выполнена: документ описывает только визуальные правила и не добавляет новые маршруты, формы, оплату, заявки, сбор данных или медицинские обещания.
- UI-правила переведены в статус утвержденных правил первой Taplink-версии.
