# План 030: Внешние кнопки видны без URL

## Статус

Завершен.

## Цель

Уточнить в `spec/feature-specs/external-links.md`, что внешние кнопки первой локальной версии должны быть видны в интерфейсе даже без подтвержденных URL.

Если URL не подтвержден, кнопка остается видимой, но должна быть неактивной и не должна выглядеть как рабочая ссылка.

## Подтверждение specs

Основание:

- подтверждение владельца проекта в текущей задаче;
- `spec/functional-map.md`;
- `spec/feature-specs/external-links.md`;
- `spec/technical-specs/change-management.md`.

Подтверждено:

- в первой версии есть кнопки `Telegram`, `WhatsApp`, `YouTube`, `Instagram`;
- владелец проекта хочет видеть эти кнопки в интерфейсе даже без URL, чтобы понимать будущий вид страницы;
- без подтвержденного URL кнопка не должна быть рабочей внешней ссылкой.

Вне specs:

- точные URL внешних платформ;
- финальный текст или визуальный стиль неактивной кнопки;
- правило открытия внешних ссылок;
- аналитика, UTM-метки, интеграции и проверка доступности.

## Что делаем

1. Обновить `spec/feature-specs/external-links.md`.
2. Зафиксировать обязательную видимость всех внешних кнопок первой версии.
3. Зафиксировать, что без URL кнопка видима, но неактивна.
4. Обновить roadmap и план после проверки.

## Что не делаем в этом плане

- Не добавляем точные URL.
- Не меняем код приложения.
- Не выбираем визуальный стиль disabled-состояния.
- Не выполняем git commit и push.

Ручная проверка:

- Убедиться, что без URL кнопки не описаны как активные ссылки.
- Убедиться, что точные URL и визуальный стиль не придуманы.

- `external-links.md` содержит правило обязательной видимости внешних кнопок.
- Неактивное состояние без URL описано без обещания перехода.
