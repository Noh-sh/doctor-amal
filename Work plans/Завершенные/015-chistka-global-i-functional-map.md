# План 015: Чистка global spec и functional map перед feature specs

## Статус

Завершен.

## Цель

Убрать противоречия между `spec/global-spec.md` и `spec/functional-map.md` перед переходом к feature specs.

## Подтверждение specs

Основание:

- Подтверждение владельца проекта в текущей задаче.
- `spec/global-spec.md`
- `spec/functional-map.md`

Подтверждено:

- Первая версия является одной локальной страницей.
- На странице есть внутренние блоки `О докторе` и `Курсы`.
- На странице есть внешние переходы.
- Блок `Курсы` может содержать названия, описания, цены и кнопку покупки с переходом в Telegram к менеджеру.
- Оформление покупки, онлайн-оплата, формы и выдача доступа внутри проекта не входят в первую локальную версию.
- Детали блоков, кнопок, ссылок и ограничений нужно раскрывать в feature specs.

Вне specs:

- Конкретные курсы, цены, ссылки и UI-реализация не добавляются.

## Что делаем

1. Уточнить `global-spec.md`, чтобы он описывал не только внешние переходы, но и блоки на этой же странице.
2. Уточнить ограничения первой версии, чтобы блок `Курсы` не конфликтовал с запретом полноценной покупки внутри проекта.
3. Уточнить финальную формулировку `functional-map.md`.
4. Добавить список тем, которые нужно детализировать в feature specs.

## Что не делаем в этом плане

- Не меняем код приложения.
- Не создаем feature specs.
- Не добавляем новые кнопки, курсы, цены или ссылки.
- Не выполняем git commit и push.

## Проверка

Команды:

```bash
rg -n "внешнюю платформу|блок|каталог курсов|покупка курсов|онлайн-оплата|feature specs|простая локальная страница ссылок" spec/global-spec.md spec/functional-map.md
git diff --check
```

Ручная проверка:

- Убедиться, что `global-spec.md` не спорит с блоками `О докторе` и `Курсы`.
- Убедиться, что `functional-map.md` не называет первую версию только страницей ссылок.
- Убедиться, что будущие детали вынесены в feature specs.

## Критерии готовности

- Верхние specs согласованы между собой.
- В первой версии разрешен блок `Курсы`, но не разрешена полноценная покупка внутри проекта.
- В `functional-map.md` указано, что нужно раскрыть в feature specs.
- Результат проверки записан в план.

## Результат проверки

Выполнены проверки:

```bash
rg -n "каталог курсов|покупка курсов|оформление и оплата|feature specs|одностраничной страницей|простая локальная страница ссылок" spec/global-spec.md spec/functional-map.md
git diff --check
```

Результат:

- `global-spec.md` теперь описывает переходы не только на внешние платформы, но и к блокам на этой же странице.
- Ограничение про курсы уточнено: в первой версии не входит отдельный каталог с внутренними страницами, корзина, оформление и оплата покупки внутри проекта.
- `functional-map.md` больше не называет первую версию простой страницей ссылок, а описывает ее как локальную одностраничную страницу с внутренними блоками и внешними переходами.
- Добавлен раздел `Что нужно детализировать в feature specs`.
- `git diff --check` выполнен без ошибок.
