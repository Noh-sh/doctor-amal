# План 092: Уточнить процесс работы агента

## Статус

Завершен.

## Цель

Уточнить `AGENTS.md` по замечанию ментора: сделать порядок работы нового AI-агента более явным и зафиксировать стиль именования новых файлов и папок на будущее.

## Подтверждение specs

Основание:

- `AGENTS.md`;
- `.agents/skills/doctor-amal-specs/SKILL.md`;
- `Work plans/TEMPLATE.md`;
- `Roadmap/TEMPLATE.md`.

Подтверждено:

- рабочий процесс проекта уже требует specs, work plans, проверки, Roadmap и отдельного подтверждения git-действий;
- можно уточнить правила процесса без изменения продуктовых требований, UI, маршрутов, данных, Supabase schema или бизнес-правил.

Вне specs:

- переименование существующих папок и файлов.

## Что делаем

1. Добавить в `AGENTS.md` явный порядок работы: specs -> активный план -> разработка -> проверка -> завершенный план -> Roadmap/changelog -> git.
2. Добавить правило именования новых файлов и папок на будущее.
3. Не переименовывать существующие папки и файлы.
4. Обновить Roadmap после проверки.

## Что не делаем в этом плане

- Не меняем продуктовые specs.
- Не меняем код приложения.
- Не переименовываем существующие `Work plans/`, `Roadmap/` и другие исторические пути.
- Не выполняем commit/push без отдельной просьбы пользователя.

## Проверка

Команды:

```bash
git diff --check
```

Ручная проверка:

- `AGENTS.md` явно описывает полный порядок работы;
- правило именования не требует переименования существующих путей.

## Критерии готовности

- `AGENTS.md` содержит уточненный порядок работы.
- `AGENTS.md` содержит future-only правило именования.
- План перенесен в `Work plans/Завершенные/`.
- `Roadmap/chronology.md` и `Roadmap/project-roadmap.md` обновлены.

## Результат проверки

- `git diff --check` выполнен без ошибок.
- `AGENTS.md` явно фиксирует порядок работы: specs -> активный план -> разработка -> проверка -> завершенный план -> Roadmap/changelog -> git.
- Правило именования добавлено только для новых файлов и папок.
- Существующие исторические пути не переименовывались.

## Измененные файлы

- `AGENTS.md`
- `Work plans/Завершенные/092-utochnit-agents-process.md`
- `Roadmap/chronology.md`
- `Roadmap/project-roadmap.md`

## Git после проверки

После успешной проверки подготовить:

```bash
git status
git add AGENTS.md Roadmap/chronology.md Roadmap/project-roadmap.md "Work plans/Завершенные/092-utochnit-agents-process.md"
git commit -m "Уточнить процесс работы агента"
```

Push на GitHub выполнять только после отдельного подтверждения пользователя.
