# План 007: Статьи

## Статус

Завершен.

## Цель

Реализовать раздел статей `/articles` и страницу отдельной статьи `/articles/[articleId]` на основе локальных данных и repository layer. Раздел должен показывать только опубликованные материалы с полным текстом, скрывать черновики и неполные статьи, обрабатывать пустое состояние и недоступную статью без технических деталей.

## Подтверждение specs

Основание:

- `spec/feature-specs/articles.md`
- `spec/technical-specs/routing-and-ui.md`
- `spec/technical-specs/data-model.md`
- `spec/technical-specs/local-storage.md`
- `spec/feature-specs/error-handling.md`
- `spec/user-stories/core-user-stories.md`
- `spec/user-stories/edge-case-stories.md`
- `spec/technical-specs/change-management.md`
- `spec/technical-specs/implementation-checklist.md`

Подтверждено:

- раздел статей должен показывать список опубликованных статей;
- карточка статьи должна показывать заголовок, краткое описание, тему, дату публикации и источник, если он указан;
- отдельная статья должна открываться по `articleId`;
- отдельная статья должна показывать заголовок, дату, тему, полный текст, источник, переход назад к списку статей и обычную навигацию к каталогу и странице "О докторе";
- черновики, архивные записи и статьи без полного текста не должны открываться как полноценные материалы;
- при пустом списке должно быть спокойное пустое состояние;
- если статья не найдена или недоступна, пользователь должен видеть понятное сообщение и переход к списку статей;
- страница статьи не должна вести напрямую к заявке на покупку;
- статьи остаются ознакомительными материалами и не заменяют очный медицинский осмотр.

Вне specs:

- поиск и фильтрация по статьям;
- админка статей;
- серверная база данных;
- публикация новых объявлений;
- комментарии, лайки, личный кабинет;
- прямой переход к заявке на покупку со страницы статьи.

## Что сделано

1. `/articles` подключен к `getPublishedArticles()`.
2. Заглушки заменены на список опубликованных материалов из локальных данных.
3. `/articles/[articleId]` подключен к `getArticleById()`.
4. Добавлено форматирование даты публикации в русском формате.
5. Добавлен безопасный вывод полного текста статьи.
6. Добавлены состояния пустого списка и недоступной статьи.
7. Черновики и опубликованные статьи без полного текста не раскрываются пользователю.
8. Добавлены стили для списка и страницы статьи в существующей визуальной системе.

## Что не делаем в этом плане

- не добавляем заявки со страницы статьи;
- не добавляем поиск, фильтры или теги как отдельный интерактивный сценарий;
- не меняем модель данных `Article`;
- не меняем seed-данные без необходимости;
- не добавляем серверное хранение или внешние интеграции.

## Проверка

Команды:

```bash
npm run build
```

Ручная проверка:

- `/articles` показывает только опубликованные статьи с полным текстом;
- `/articles/skin-and-lifestyle` открывает опубликованную статью;
- `/articles/draft-short-note` не раскрывает черновик;
- `/articles/incomplete-published-note` не раскрывает статью без полного текста;
- `/articles/unknown` показывает понятное состояние недоступной статьи;
- страница статьи не содержит перехода к заявке на покупку.

## Критерии готовности

- раздел статей работает через repository layer;
- черновики и неполные статьи скрыты;
- опубликованная статья открывается по URL;
- недоступная статья не раскрывает содержимое;
- пользовательские тексты на русском языке и сохраняют медицинские ограничения;
- `npm run build` выполнен успешно.

## Результат проверки

- `npm run build` выполнен успешно.
- Production server из собранного приложения запущен на `http://localhost:3001` для проверки и остановлен после проверки.
- Через HTTP-проверку получен `200 OK` для `/articles`.
- Через HTTP-проверку получен `200 OK` для `/articles/skin-and-lifestyle`.
- Через HTTP-проверку получен `200 OK` для `/articles/draft-short-note`; страница показывает безопасное состояние "Статья не найдена или пока недоступна" и не раскрывает черновик.
- Через HTTP-проверку получен `200 OK` для `/articles/incomplete-published-note`; страница показывает безопасное состояние и не раскрывает материал без полного текста.
- В HTML опубликованной статьи проверено отсутствие прямого перехода к заявке на покупку.
- Dev server на `3000` не использовался для проверки: Next сообщил о существующем dev-lock с PID `20254`, но HTTP-запрос к `localhost:3000` из текущей среды не подключился.
