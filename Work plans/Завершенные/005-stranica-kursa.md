# План 005: Страница курса

## Статус

Завершен.

## Цель

Реализовать страницу `/courses/[courseId]`, которая показывает подробную информацию о опубликованном курсе и доступные действия в зависимости от статуса продаж.

## Подтверждение specs

Основание:

- `spec/feature-specs/course-detail-page.md`
- `spec/technical-specs/routing-and-ui.md`
- `spec/technical-specs/data-model.md`
- `spec/feature-specs/error-handling.md`
- `spec/user-stories/core-user-stories.md`
- `spec/user-stories/edge-case-stories.md`
- `spec/technical-specs/local-storage.md`
- `spec/technical-specs/change-management.md`
- `spec/technical-specs/implementation-checklist.md`

Подтверждено:

- страница курса открывает курс по `courseId`;
- пользователь видит только опубликованный курс;
- страница показывает название, полное описание, категорию, темы, симптомы, пользу, стоимость, статус продаж, период продаж, условия покупки и Telegram-доступ;
- страница объясняет, что оплата не выполняется на сайте и доступ в Telegram предоставляется вручную после согласованной оплаты;
- при `salesStatus = "open"` нужно показать переход к заявке на покупку и консультации по подбору курса;
- при `salesStatus = "closed"` нельзя показывать заявки, нужно предложить статьи и указать, что перед следующим открытием продаж будет объявление;
- при `salesStatus = "coming_soon"` нельзя принимать заявку, нужно показать период, если он известен, и предложить статьи;
- блок похожих курсов можно показать, если есть опубликованные связанные курсы, и он должен вести на страницы курсов;
- технический `id`, внутреннее название Telegram-канала и закрытые материалы курса не показываются;
- если курс не найден или не опубликован, пользователь видит понятное состояние и переход в каталог.

Вне specs:

- нет.

## Что делаем

1. Проверить текущие модели, repository layer, правила продаж и данные курсов.
2. Подключить страницу `/courses/[courseId]` к repository layer.
3. Показать подробные данные курса без служебных полей.
4. Реализовать действия для статусов `open`, `closed`, `coming_soon`.
5. Добавить блок похожих опубликованных курсов, если они есть.
6. Добавить понятное состояние для несуществующего или недоступного курса.
7. Обновить стили в рамках существующей визуальной системы.
8. Выполнить техническую проверку и ручные сценарии.
9. После проверки перенести план в завершенные и обновить roadmap.

## Что не делаем в этом плане

- Не реализуем форму заявки.
- Не сохраняем заявки.
- Не добавляем онлайн-оплату.
- Не добавляем автоматическое добавление в Telegram.
- Не раскрываем закрытые материалы курса.
- Не добавляем личный кабинет.
- Не добавляем серверный API.
- Не меняем модель данных и seed-данные без необходимости.
- Не меняем `spec/`.

## Проверка

Команды:

```bash
npm run build
```

Ручная проверка:

- открыть страницу опубликованного курса со статусом `open`;
- проверить, что есть переход на `/request?type=course_purchase&courseId=...`;
- проверить, что есть переход на `/request?type=pre_purchase_consultation`;
- открыть курс со статусом `closed`;
- проверить, что заявки не предлагаются и есть переход к статьям;
- открыть курс со статусом `coming_soon`;
- проверить, что заявки не предлагаются и период показан, если заполнен;
- открыть несуществующий курс;
- проверить, что есть понятное сообщение и переход в каталог;
- проверить, что внутреннее название Telegram-канала и технический `id` не выводятся.

## Критерии готовности

- `/courses/[courseId]` показывает опубликованный курс по `courseId`.
- Черновой или несуществующий курс не раскрывает содержимое.
- Страница объясняет отсутствие онлайн-оплаты и ручной Telegram-доступ.
- Действия соответствуют статусу продаж.
- Похожие курсы ведут на страницы курсов.
- Build проходит.
- Результат проверки записан в план.
- `Roadmap/chronology.md` и `Roadmap/project-roadmap.md` обновлены после завершения.

## Результат проверки

Техническая проверка выполнена 2026-05-16.

- Страница `/courses/[courseId]` подключена к `getCourseById()` и `getPublishedCourses()`.
- Опубликованный курс со статусом `open` показывает подробные данные, ссылку на заявку покупки и ссылку на консультацию по подбору курса.
- Курс со статусом `closed` не показывает ссылки на заявки и предлагает перейти к статьям.

- Несуществующий курс показывает понятное сообщение и переход в каталог.
- Черновой курс не раскрывает пользовательский заголовок.
