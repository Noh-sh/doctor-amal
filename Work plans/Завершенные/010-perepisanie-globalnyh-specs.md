# План 010: Переписывание глобальных specs под Taplink-формат

## Статус

Завершен.

## Цель

Обновить верхний уровень спецификаций проекта Doctor Amal под новую подтвержденную концепцию: локальная web-страница в формате Taplink со ссылками на внешние платформы доктора.

## Подтверждение specs

Основание:

- Подтверждение владельца проекта в текущей задаче.
- `spec/global-spec.md`
- `spec/functional-map.md`
- `spec/technical-specs/change-management.md`

Подтверждено:

- Проект предназначен для русскоязычной аудитории женского пола любых возрастов, которая умеет читать и пользоваться интернетом.
- Проект должен работать как Taplink-страница: пользователь нажимает кнопки и переходит на нужные внешние платформы.
- Первая версия является локальной.
- Проект не обещает выздоровление и не заменяет очную консультацию.
- `global-spec.md` должен фиксировать общий смысл проекта.
- `functional-map.md` должен подробно разложить кнопки, платформы, внутренние страницы, медицинские тексты, сбор данных и бизнес-цель.

Вне specs:

- Точный список всех кнопок первой версии, кроме Telegram, еще не подтвержден.
- Реализация UI и кода в этом плане не выполняется.

## Что делаем

1. Переписать `spec/global-spec.md` под новую концепцию.
2. Переписать `spec/functional-map.md` как функциональную карту Taplink-страницы.
3. Проверить, что верхние specs больше не описывают старую логику каталога курсов и локальных заявок как текущую первую версию.

## Что не делаем в этом плане

- Не меняем код приложения.
- Не переписываем feature specs и technical specs.
- Не добавляем конкретные кнопки, которые владелец проекта еще не подтвердил.
- Не выполняем git commit и push.

## Проверка

Команды:

```bash
rg -n "каталог курсов|заявк|покупк|открытых продаж|Taplink|личные данные" spec/global-spec.md spec/functional-map.md
```

Ручная проверка:

- Убедиться, что `global-spec.md` отвечает на вопросы: кто, где, для чего, что делает и чего не делает проект.
- Убедиться, что `functional-map.md` описывает первую локальную версию как страницу ссылок.
- Убедиться, что неподтвержденные кнопки не выданы за утвержденные требования.

## Критерии готовности

- Верхние specs соответствуют новой концепции проекта.
- В specs явно указано, что первая версия локальная.
- В specs явно указано, что первая версия не обещает выздоровление и не заменяет очную консультацию.
- В specs явно указано, что первая версия не собирает личные данные, пока отдельное требование не подтверждено.
- Результат проверки записан в план.

## Результат проверки

Выполнена проверка:

```bash
rg -n "каталог курсов|заявк|покупк|открытых продаж|Taplink|личные данные|Telegram|выздоровление|очная консультация" spec/global-spec.md spec/functional-map.md
```

Результат:

- `Taplink`, `Telegram`, `личные данные`, `выздоровление` и `очная консультация` явно отражены в новых верхних specs.
- Старая логика каталога курсов, покупки и заявок оставлена только в разделах ограничений первой версии.
- Упоминаний открытых продаж как поведения текущей первой версии не осталось.
- Неподтвержденные кнопки не выданы за утвержденные требования.
