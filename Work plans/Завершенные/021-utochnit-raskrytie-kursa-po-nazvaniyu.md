# План 021: Уточнить раскрытие курса по названию

## Статус

Завершен.

## Цель

Уточнить `courses-block.md`: в блоке `Курсы` сначала показываются названия курсов, а описание, цена и кнопка `Купить` раскрываются после нажатия на название курса.

## Подтверждение specs

Основание:

- Подтверждение владельца проекта в текущей задаче.
- `spec/functional-map.md`
- `spec/feature-specs/courses-block.md`

Подтверждено:

- В блоке `Курсы` сначала показываются названия курсов.
- При нажатии на название курса раскрываются описание курса, цена и кнопка `Купить`.
- При нажатии на `Купить` пользователь сразу переходит в Telegram для связи с менеджером.

Вне specs:

- Конкретные курсы, цены, валюта и Telegram-ссылка пока не подтверждены.

## Что делаем

1. Обновить раздел `Что пользователь видит`.
2. Обновить действия пользователя.
3. Обновить основное поведение раскрытия курса.
4. Сохранить запрет на отдельные маршруты, формы, оплату и сбор данных.

## Что не делаем в этом плане

- Не добавляем реальные курсы, цены и ссылки.
- Не меняем код приложения.
- Не выполняем git commit и push.

## Проверка

Команды:

```bash
rg -n "название курса|название выбранного курса|описание курса|цена|Купить|Telegram|раскры|нажат" spec/feature-specs/courses-block.md
git diff --check
```

Ручная проверка:

- Убедиться, что описание, цена и кнопка `Купить` появляются после нажатия на название курса.
- Убедиться, что `Купить` ведет в Telegram к менеджеру.

## Критерии готовности

- Сценарий раскрытия курса по названию зафиксирован.
- Первая локальная версия не расширена формами, оплатой или сбором данных.
- Результат проверки записан в план.
