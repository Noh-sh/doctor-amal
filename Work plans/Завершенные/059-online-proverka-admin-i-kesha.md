# План 059: Online-проверка админки и кеша

## Статус

Завершен.

## Цель

Проверить online-версию после merge: публичную страницу, вход в `/admin`, сохранение контента через Supabase и обновление публичной страницы примерно за 30 секунд.

## Подтверждение specs

Основание:

- `spec/technical-specs/admin-auth-and-access.md`
- `spec/technical-specs/supabase-content-source.md`
- `spec/technical-specs/change-management.md`

Подтверждено:

- публичная страница `/` читает опубликованный контент из Supabase;
- `/admin` доступен только доктору через email/password и роль `doctor_admin`;
- админка может менять подтвержденные поля публичного контента;
- публичная страница может обновляться не мгновенно из-за кеша чтения Supabase;
- текущий delay кеша сокращен до 30 секунд.

Вне specs:

- нет.

## Что делаем

1. Проверить, что online deploy доступен.
2. Проверить публичную страницу `/`.
3. Проверить, что `/admin` открывает форму входа.
4. Войти доктором через email/password.
5. Изменить безопасное тестовое поле контента, например описание существующего курса.
6. Сохранить изменение и проверить, что публичная страница показывает новое значение в течение примерно 30 секунд.
7. Проверить, что после чистой перезагрузки `/admin` нет warning `Multiple GoTrueClient instances`.
8. Вернуть тестовое поле к нужному финальному значению, если для проверки использовалась временная правка.
9. Если feedback сохранения непонятен, добавить явный локальный индикатор `Сохранено` с галочкой у сохраненной формы.

## Что не делаем в этом плане

- Не добавляем новые курсы.
- Не удаляем курсы.
- Не меняем RLS, Auth users, migrations, env-переменные или medical notice.
- Не добавляем мгновенный сброс кеша после сохранения.
- Не меняем публичный сценарий и не добавляем ссылки на `/admin` на публичной странице.
- Не добавляем журнал изменений, черновики или предпросмотр публикации.

## Проверка

Автоматические команды:

```bash
git status --branch --short
npm run build
git diff --check
```

Ручная online-проверка:

- `/` открывается без авторизации;
- `/admin` показывает форму входа;
- доктор входит через email/password;
- доктор сохраняет изменение существующего курса;
- после сохранения у формы появляется понятный индикатор `Сохранено`;
- `/` показывает изменение примерно за 30 секунд;
- обычный публичный сценарий не меняется;
- warning `Multiple GoTrueClient instances` не появляется после чистой перезагрузки `/admin`.

## Критерии готовности

- Online-страница и online-админка работают после merge.
- Сохранение через админку попадает в Supabase.
- Публичная страница обновляется примерно в рамках 30 секунд.
- Если выявлены проблемы, они зафиксированы отдельно и не маскируются изменением specs.
