# Планы работ

Эта папка хранит рабочую историю проекта Doctor Amal.

## Структура

- `Активные/` - планы, которые сейчас находятся в работе.
- `Завершенные/` - планы, которые уже выполнены, проверены и готовы к фиксации в git.

## Правило работы

1. Перед началом задачи создать или обновить файл в `Work plans/Активные/`.
2. Для нового плана использовать шаблон `Work plans/TEMPLATE.md`.
3. Работать только по требованиям из `spec/`.
4. После завершения части работы выполнить проверку, подходящую риску изменения.
5. Записать результат проверки в файл активного плана.
6. После успешной проверки переместить файл из `Work plans/Активные/` в `Work plans/Завершенные/`.
7. Обновить хронологию в `Roadmap/chronology.md`.
8. После этого подготовить git-команды для локального commit.
9. Commit выполнять только после подтверждения пользователя.
10. Push на GitHub выполнять только после отдельного подтверждения пользователя.

## Важно

- Эта папка не заменяет `spec/`.
- Новые функции, маршруты, поля данных, бизнес-правила и пользовательские сценарии сначала должны быть подтверждены в `spec/`.
- Git commit выполняется только после проверки и подтверждения пользователя.
- Push на GitHub выполняется только после отдельного подтверждения пользователя.

## Что записывать в завершенный план

Перед переносом плана в `Work plans/Завершенные/` нужно записать:

- что было сделано;
- какие проверки выполнены;
- результат проверок;
- какие файлы изменены;
- какие git-команды подготовлены или выполнены после подтверждения пользователя.
