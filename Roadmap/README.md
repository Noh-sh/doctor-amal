# Roadmap

Эта папка хранит хронологическую историю развития проекта Doctor Amal по областям работы.

## Назначение

`Roadmap` нужен, чтобы после завершения каждого плана было видно:

- какая общая стадия проекта сейчас актуальна;
- что изменилось в проекте;
- к какой области относится работа;
- какие проверки были выполнены;
- что осталось следующим шагом.

## Файлы

- `project-roadmap.md` - общая дорожная карта: что уже сделано, что следующее, что запланировано.
- `chronology.md` - хронологическая история завершенных действий.
- `TEMPLATE.md` - шаблон записи для хронологии.

## Правило обновления

После завершения плана из `Work plans/Активные/` нужно:

1. записать результат проверки в файл плана;
2. перенести план в `Work plans/Завершенные/`;
3. добавить запись в `Roadmap/chronology.md` по шаблону `Roadmap/TEMPLATE.md`;
4. обновить статусы в `Roadmap/project-roadmap.md`;
5. только после этого готовить локальный git commit;
6. commit выполнять только после подтверждения пользователя;
7. push на GitHub выполнять только после отдельного подтверждения пользователя.

## Области работ

Рекомендуемые области:

- `Процесс` - рабочие правила, планы, организация задач;
- `Frontend` - страницы, компоненты, маршруты, клиентское поведение;
- `Дизайн` - визуальная структура, адаптивность, стили, UI-состояния;
- `Данные` - модели, JSON, repository layer;
- `Заявки` - формы, валидация, `localStorage`, защита от повторной отправки;
- `Контент` - пользовательские тексты, статьи, описание курсов;
- `Проверка` - ручные сценарии, build, lint, приемка;
- `Документация` - рабочие документы, README, правила проекта.

Если работа затрагивает несколько областей, нужно указать все.
